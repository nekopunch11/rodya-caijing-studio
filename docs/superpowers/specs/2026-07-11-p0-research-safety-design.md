# 财经内容台 P0 研究安全修复设计

日期：2026-07-11  
状态：用户已确认  
范围：仅 P0，不包含七模块 P1 方法论扩展

## 目标

把 `rodya-caijing-studio` 从“框架专业但部分规则可能制造伪精确”修复为“关键结论有证据门槛、关键计算可复核、数据不足时主动收窄结论”的研究工具。保持现有七模块路由、三形态渲染和港股打新自用/对客双模式。

## 变更边界

### 1. 港股打新

- 删除 `公开发售手数 ÷（超购倍数 × 估算申购人数）` 及其固定阈值，不再用重复计入需求的公式判断“一人一手”。
- 只有取得招股书发售结构、公开发售手数、实际或可比申请档位与配售结果时，才估算档位中签率和期望获配；缺任一关键输入时只给区间、条件性判断和补数清单。
- 将固定 40/30/20/10 的“贝叶斯加权”改为透明的“证据评分/情景权重”。未经训练、样本外验证和概率校准，不输出暗盘、首日或多期限精确破发概率。
- 删除固定“破发 10% 为最坏情况”。压力测试改用同类样本的经验分位；样本不足时至少输出常规、压力、极端三档，并说明仅为情景而非概率预测。
- 将“边际成本约为零就应打满”改为五项共同约束：边际期望收益为正、最大可承受损失、抵押/融资安全、流动性与执行风险、组合资金冲突。任一不满足，不得推荐打满。
- 保留自用模式明确推荐，但必须写成“基于当前输入与约束的条件性方案”；删除固定分钟级下单与机械清仓表，改为按价格、流动性、获配数量和风险预算触发的条件清单。
- 补入 HKEX 规则：18C 特专科技公司使用其指定公开发售分配机制，不得按普通发行人自由采用机制 B；执行时仍须核对最新规则。

### 2. 专业版完成标准

- 删除所有 4000/5000 字硬下限。
- 保留“默认完整专业版”，完成标准改为：规定分析块覆盖、关键结论证据覆盖、关键数字来源与截至日、公式输入与假设透明、结论—证据—证伪闭环、数据缺口显著披露。
- 数据不足时仍交付完整结构，但不允许靠扩写补足篇幅；不适用项写明原因和替代观察指标。

### 3. 研究与合规安全 gate

新增共享 reference，供父 skill 和七模块按需读取：

- 先识别个人自用、持牌/机构内部研究、对客材料三种场景。
- 个人自用允许研究判断；涉及具体资金与操作时必须先取得风险预算、融资条件和最大损失承受度。
- 机构内部研究保留专业判断，但必须保留来源、方法、假设、利益冲突和版本留痕。
- 对客材料继续执行合规降级，并增加所属机构复核、适当性、利益冲突与留痕提示。
- 固定免责声明只作风险提示，不能替代牌照、机构制度、适当性或合规审查。

## 测试设计

先写契约测试并在旧规则上确认失败，再修改生产文件。测试至少断言：

1. 错误中签压力公式及固定阈值不存在。
2. 未校准模型不得称为贝叶斯、不得输出精确破发概率。
3. “固定 -10% 最坏情况”和无条件“应打满”不存在。
4. 18C 机制限制、安全 gate 和缺数降级规则存在。
5. 4000/5000 字硬下限在生产规则中不存在。
6. 专业版完整度契约、对客机构复核和免责声明边界存在。
7. 根 skill、子模块和共享层不存在互相冲突的旧规则。
