# 财经内容台 PPT 通用高级视觉系统设计规范

日期：2026-07-10
状态：用户已批准设计方向，待书面规范复核后进入实施计划
适用项目：`rodya-caijing-studio`

## 一、设计结论

财经内容台的按需 PPT 默认采用 **A 版：投顾内部研究汇报**。高级感来自稳定的排版系统，而不是固定的行业配色或单一图片题材。

系统固定以下能力：

- 研究命题式标题；
- MiSans + IBM Plex Mono 字体体系；
- `1+3` 数据层级；
- 十二列网格、发丝线、直角布局；
- 可控留白与数据台密度；
- 七类内部研究版式；
- 内容驱动的图片角色、图片比例与降级规则；
- 自适应主题令牌；
- Codex 生图增强与 Claude 无图回退；
- 真 `.pptx` 与 HTML deck 同构。

系统不固定冷银、铜金、蓝色、行业色、实验室、机房、工厂或任何单一视觉隐喻。不同股票可以有不同主题；同一套 deck 内必须保持一致。

## 四、渲染流程

```text
结构化分析内核
  → 选择内部研究专业口径
  → 提炼研究命题和核心矛盾
  → 规划页面叙事
  → 选择 deck 主题模式和唯一 accent
  → 判断每页是否需要图片
  → 可用时生成 / 读取图片资产
  → 渲染真 .pptx
  → 同构渲染 HTML deck
  → 有图 / 无图双路径 QA
```

所有页面先由分析内核驱动。图片不得反向决定结论、数据或研究结构。

## 五、设计系统

### 5.1 设计参数

- `DESIGN_VARIANCE: 6`：不对称但受网格约束。
- `MOTION_INTENSITY: 1`：真 PPT 静态；HTML 只保留翻页和必要转场。
- `VISUAL_DENSITY: 8`：高信息密度，以发丝线和层级组织，不使用卡片堆叠。

### 5.2 网格

- 画布：16:9。
- 基础网格：十二列。
- 左右安全边距：画布宽度的 4%–6%。
- 顶部功能区：画布高度的 7%–12%。
- 底部来源区：画布高度的 4%–7%。
- 内容间距使用统一节奏，不因图片有无改变标题、页脚和数据基线。
- 直角系统，圆角默认为 0。
- 发丝线只承担分组、对齐和坐标功能，不作空洞装饰。

### 5.3 留白与密度

留白用于建立层级，不用于制造海报式空洞。

- 内容页优先占据有效内容区的 70%–85%。
- 普通内容页不得出现无意义的集中空白带。
- 单块无信息留白通常控制在 10%–16%。
- 封面和章节页允许为标题或图片保留更大的负空间，但必须承担构图功能。
- 内容超量时依次处理：删低价值文字 → 缩减或取消图片 → 拆页。
- 禁止通过缩小正文字号硬塞内容。

### 5.4 不使用通用卡片 UI

- 不采用三张或四张等宽圆角卡片作为默认信息容器。
- 数据使用原生网格、列、发丝线和大小层级。
- 只有需要明确独立语义的假设框、风险框可以使用边界容器。
- 阴影不是层级工具；PPT 主体不使用网页式浮层阴影。

## 六、字体系统

### 6.1 字体角色

| 角色 | 首选 | 回退 |
| --- | --- | --- |
| 中文标题 / 正文 | MiSans / MiSans VF | Microsoft YaHei / PingFang SC / sans-serif |
| 数字 / 英文 / 股票代码 | IBM Plex Mono | JetBrains Mono / Consolas / Courier New |

### 6.2 字号建议

| 角色 | 建议字号 |
| --- | --- |
| 封面研究命题 | 40–54pt |
| 页面标题 | 30–36pt |
| 章节大字 | 36–48pt |
| 正文 / 结论 | 16–20pt |
| 数据主数字 | 40–72pt |
| 辅助数字 | 22–34pt |
| 标签 / 来源 / 页码 | 9–12pt |

### 6.3 字体纪律

- 所有数字使用 tabular 对齐。
- 标题通过字重、对比和位置建立层级，不依赖超大字号。
- 英文和数字不使用夸张终端字距。
- 中文正文不使用等宽字体。
- 真 `.pptx` 需写明字体回退；有嵌入能力时优先嵌入免费字体。

## 七、自适应主题系统

### 7.1 固定功能令牌

```yaml
theme:
  mode: dark | light | media-led
  surface: 页面底色
  surface_alt: 次级底色
  ink: 主要文字
  muted: 辅助文字
  hairline: 发丝线
  accent: deck 唯一点睛色
  up: 涨 / 正向变化
  down: 跌 / 负向变化
  scrim: 图片文字遮罩
```

### 7.2 三种主题模式

1. **深色主题**：适合需要低明度、强对比或暗色图片的 deck。
2. **浅色主题**：适合高明度品牌、消费、医药、服务或浅色图片。
3. **图片主导主题**：从主视觉或用户品牌中取得一个低饱和度 `accent`，其余令牌保持中性。

三种模式是结构模式，不绑定行业。工业公司可以使用浅色主题，消费公司也可以使用深色主题。

### 7.3 accent 选择顺序

1. 用户明确指定的品牌色；
2. 用户提供的品牌资产；
3. deck 主视觉中的低饱和度颜色；
4. 行业主题色；
5. 无可靠依据时使用中性 accent。

### 7.4 颜色纪律

- 同一套 deck 只有一个 `accent`。
- `accent` 只用于关键线、章节识别、一个主数字或核心标记，占画面比例约 5% 以内。
- `up/down` 永久保留给有方向含义的数据，不作装饰。
- 图片可以保留自身颜色，但需通过裁切、遮罩、降低饱和度或统一明度融入主题。
- deck 内主题锁定，不允许逐页更换色彩语言。
- 不锁定冷银、铜金、蓝色或任何固定色板。

## 八、研究命题与文案层级

### 8.1 标题结构

封面和模块页优先采用研究命题，不使用泛化报告名占据主标题。

```text
标的 / 代码
研究命题（主标题）
分析范围 / 数据截至日
核心判断
```

示例中的“某公司综合体检”可以作为文件名、眉题或副标题，但不能自动成为最大标题。

### 8.2 内部研究语言

允许使用：

- 专业判断；
- 核心矛盾；
- 逻辑支柱；
- 情景与敏感度；
- 估值消化；
- 证伪条件；
- 跟踪重点。

不得使用空泛营销词、收益承诺或无证据的确定性语气。

### 8.3 微标签纪律

- “内部研究”全页只出现一次。
- 不同时堆叠英文 badge、中文口径、版本号和状态标签。
- 元数据优先用列对齐或发丝线分组。
- 中点分隔符不连续堆叠；需要多个字段时用列或换行。
- 页码是功能件，允许保留。

## 九、`1+3` 数据层级

### 9.1 默认结构

- `1`：一个最能支撑本页结论的主指标。
- `3`：最多三个辅助指标，提供对照坐标、质量或风险解释。

每个数字必须满足：值 + 对照坐标 + 来源 + 截至日 / 口径。

### 9.2 例外

以下版式不强制 `1+3`：

- 总览表；
- 风险与证伪清单；
- 产业链 Pipeline；
- 同业比较表；
- 免责声明。

例外页面仍需有一个明确结论，不得成为无主次的数据堆。

### 9.3 图表纪律

- 财务数字、估值坐标、时间序列和比较关系必须用原生 PPT 图表、形状或表格绘制。
- AI 生图不得生成图表、数字或金融结论。
- 当前点、分位、破发率等统计信息使用主题 `ink/accent`，只有真实方向变化才使用 `up/down`。

## 十、七类内部研究版式

### 10.1 研究命题封面

- 标的和代码为小层级；
- 研究命题为最大标题；
- 一句核心判断；
- 可使用图片，占画面约 35%–50%；
- 图片缺失时改为非对称纯排版，不保留空图片槽。

### 10.2 综合结论总览

- 汇总模块结论、主数字、确定性与数据缺口；
- 使用账本表或非对称分组；
- 默认不使用图片。

### 10.3 投资逻辑与核心矛盾

- 1 个研究命题；
- 2–4 个逻辑支柱；
- 每个支柱带可观测指标和证伪条件；
- 图片可选，占比不超过约 30%。

### 10.4 基本面 / 财报模块页

- 默认 `1+3` 数据层级；
- 一段专业判断；
- 跟踪变量；
- 图片通常为 0%–25%。

### 10.5 估值与同业坐标页

- 主数字为估值分位、隐含增速或关键估值坐标；
- 配合区间、同业或历史坐标；
- 默认不使用装饰图片。

### 10.6 风险、证伪与跟踪页

- 按风险维度组织；
- 风险 → 当前状态 → 缓释因子 → 可观测指标；
- 结论分轻重；
- 默认不使用图片。

### 10.7 免责声明页

- 保持简洁；
- 免责声明、数据截至日、来源口径完整；
- 可继承封面主题，但不使用新的视觉题材。

### 10.8 可选 Pipeline 页

产业链项目可在版式库中插入 Pipeline 页面。它不是固定第几页，但需保持上游 / 中游 / 下游、利润池、瓶颈点和代表公司的清晰关系。

## 十一、内容驱动的图片系统

### 11.1 图片不是默认必需件

只有图片能改善叙事、行业理解或视觉节奏时才使用。以下情况直接使用无图版式：

- 数据密度高；
- 图片不能提供新增信息；
- 找不到或生成不出高质量图片；
- 图片会挤压结论、数字、风险或来源；
- 图片可能造成公司事实误认。

### 11.2 图片角色

| 内容 | 可选视觉角色 |
| --- | --- |
| 公司综合分析 | 业务场景、产品类别、基础设施、材料或抽象主题 |
| 财报 | 产品 / 服务场景、业务结构、无图数据页 |
| 估值 | 通常无图；必要时仅用低干扰抽象背景 |
| 风险 | 默认无图 |
| 产业链 | 原材料、生产过程、结构关系、基础设施 |
| A股 / 港股新股 | 业务场景、产品类别、行业基础设施；不使用金币或交易屏 |
| 消费行业 | 产品、空间、生活方式、材料微距 |
| 医药行业 | 研发、实验、生产、医疗场景；不可默认使用实验室 |
| 金融服务 | 服务场景、基础设施、空间和抽象秩序；避免城市股票屏 |

### 11.3 图片比例

| 页面职责 | 图片占比 |
| --- | --- |
| 封面 | 35%–50% |
| 章节页 | 40%–60% |
| 公司 / 产业链内容页 | 25%–40% |
| 基本面 / 财报页 | 0%–25% |
| 估值 / 风险数据页 | 0%–20%，默认无图 |

信息优先级：研究命题 > 核心判断 > 主数字 > 对照坐标 > 风险 / 证伪 > 来源 / 截至日 > 图片。

### 11.4 生图提示词变量

```yaml
visual_prompt:
  domain: 行业与业务属性
  thesis: 页面研究命题
  role: 图片承担的叙事作用
  placement: 图片位置和裁切比例
  negative_space: 文字负空间位置
  theme_mode: dark | light | media-led
  accent_hint: 可选 deck accent
  realism: 真实材质、自然光、适度不完美
```

### 11.5 生图硬限制

- 不生成文字、数字、Logo、股票代码、图表、金融结论或水印。
- 不虚构公司厂房、产品外观、管理层、门店或办公地点。
- 无法核实具体场景时，只生成行业主题视觉，不伪装成公司实景。
- 禁止握手、金币、上涨箭头、K 线屏幕、城市股票屏等廉价财经隐喻。
- 禁止默认赛博霓虹、实验室、机房或完美 CGI。
- 只有行业和研究命题需要时才使用科技感或实验环境。
- 加入真实材质变化、自然光、轻微使用痕迹和编辑摄影质感。
- 每套 deck 只使用一个图片风格家族。

## 十二、Codex / Claude 双能力路径

### 12.1 共用路径

两端都执行：

1. 同一结构化分析内核；
2. 同一研究命题；
3. 同一页面规划；
4. 同一主题令牌；
5. 同一版式和图片比例；
6. 同一数据、来源和口径；
7. 同一真 `.pptx` / HTML 输出契约。

### 12.2 Codex 生图增强

- 仅在图片决策为“需要”且有 imagegen 能力时生成。
- 先生成图片资产，再把本地路径交给 renderer。
- renderer 不直接依赖 imagegen API。
- 图片未通过质量检查时降级为无图版式。

### 12.3 Claude 无图回退

- 使用用户提供图片；或
- 使用高级纯排版版式；或
- 使用原生图表 / 形状表达信息。

不得保留空白图片框、低质量占位图或“图片生成失败”提示。

## 十三、真 PPT 与 HTML 同构

### 13.1 共用输入

两套模板接受同一份页面数据：

```yaml
deck:
  title: 研究命题
  subject: 标的 / 代码 / 市场
  as_of: 数据截至日
  theme: 主题令牌
  slides:
    - type: layout_type
      conclusion: 本页结论
      hero_metric: 可选主指标
      support_metrics: 可选辅助指标
      evidence: 数据与来源
      watchlist: 跟踪 / 证伪
      visual_asset: 可选本地图片路径及裁切信息
```

### 13.2 renderer 职责

- `deck-template.mjs` 用 pptxgenjs 生成可编辑 `.pptx`。
- `template-deck.html` 使用同一结构、主题和图片裁切策略。
- 图片不存在时，两个 renderer 都必须切换到对应无图布局。
- 不允许两套模板各写一份不同的分析内容。

## 十四、失败处理

| 情况 | 处理 |
| --- | --- |
| imagegen 不可用 | 使用无图或用户供图路径 |
| 图片质量低 | 弃图并重新排版，不把低质量图片塞入 PPT |
| 图片与公司事实可能混淆 | 改为抽象行业视觉或无图 |
| 字体缺失 | 使用明确回退链，记录但不阻塞 |
| 内容过多 | 删低价值内容、取消图片或拆页 |
| 页面溢出 | 调整结构或拆页，不缩小到不可读字号 |
| `.pptx` 与 HTML 不一致 | 以结构化内核和 `ppt-visual-spec.md` 为准修复两端 |

## 十五、测试与验收

### 15.1 静态契约

- `ppt-visual-spec.md` 为唯一视觉事实源；
- PPT 仍只在用户明确点名时生成；
- 真 `.pptx` 与 HTML deck 版式类型一致；
- 有图和无图路径均完整；
- 图片缺失不会留下空槽；
- `up/down` 只用于方向数据；
- 一套 deck 只有一个 `accent`；
- “内部研究”每页最多一次；
- 不硬编码样例公司和样例图片。

### 15.2 七页样张

至少生成：

1. 研究命题封面；
2. 综合结论总览；
3. 投资逻辑与核心矛盾；
4. 基本面 / 财报模块页；
5. 估值与同业坐标页；
6. 风险、证伪与跟踪页；
7. 免责声明页。

有产业链内容时另测 Pipeline 页。

### 15.3 双路径样张

- 一套有图样张；
- 一套无图回退样张；
- 两套结构、数据、标题和来源完全一致；
- 差异只在 `visual_asset` 和由此触发的版式空间分配。

### 15.4 视觉 QA

- 每页单独渲染为 PNG 并逐页检查；
