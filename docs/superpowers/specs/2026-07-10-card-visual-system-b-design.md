## 一、设计结论

客户卡片统一采用方案 B「机构主视觉 + 墨账本数据系统」。取消 A/B 两套视觉之间的运行时选择，方案 B 成为唯一客户卡片视觉母体。

方案 B 不等于每张卡都必须有图片。它由六种信息构图组成，其中只有主视觉结论页和证据墙需要图片；其余版式以高密度数据、对比、风险和产业链结构为主体。图片是可降级组件，重要信息是不可降级组件。

专业版 docx 的炭黑金研报规范、客户版按需规则、合规红线、一份结构化分析内核和零依赖 HTML 交付原则均保持不变。

## 二、目标与非目标

### 目标

1. 让七个财经模块拥有统一、可信、可识别的客户卡片语言。
2. 在微信、小红书和手机截图场景中兼顾信息密度与第一眼停留。
3. Claude 与 Codex 共用同一套 HTML、数据字段和合规规则。
4. 让 Codex 的生图能力只增强主视觉，不接管文字、数据和图表排版。
5. 在没有高质量图片时仍能稳定生成完整、专业的纯数据卡片。

### 非目标

1. 本轮不设计 PPT 母版。
2. 本轮不制作小红书运营选题和完整内容包。
3. 本轮不加入 Live Photo、视频或动效。
4. 本轮不恢复 Node、Playwright 或其他卡片渲染依赖。
5. 本轮不修改研究方法论、公式、阈值或合规规则。

## 三、设计判断

这是面向个人投资者、券商投顾和客户经理的受监管财经视觉系统。视觉语言必须首先传递可信、克制、专业和证据意识，其次才是社交平台的吸引力。

设计参数：

| 参数 | 数值 | 解释 |
| --- | ---: | --- |
| DESIGN_VARIANCE | 5 | 构图可变化，但品牌骨架稳定 |
| MOTION_INTENSITY | 1 | 当前为静态卡片，不加入装饰动效 |
| VISUAL_DENSITY | 8 | 财经信息优先，使用网格和发丝线组织密度 |

## 四、视觉母体

### 4.1 品牌令牌

| 角色 | 值 | 使用规则 |
| --- | --- | --- |
| 品牌墨黑 | `#111111` | 结构、招牌带、页脚、主文字 |
| 纸白 | `#FDFDFB` | 主信息底色 |
| 冷灰底 | `#F2F2EF` | 结论框、次级信息区 |
| 发丝线 | `#DEDEDA` | 网格、分隔和表格 |
| 涨红 | `#C0392B` | 仅表示上涨、正增长、超预期方向 |
| 跌绿 | `#1E9E5A` | 仅表示下跌、负增长、不及预期方向 |
| 次级文字 | `#6B6B68` | 元数据、来源和口径 |

红绿不得进入品牌色块、招牌带、装饰背景或无方向含义的统计值。估值分位、破发率、风险等级和当前点默认使用墨色。

### 4.2 字体与数字

- 中文标题：MiSans、思源黑体或系统无衬线回退。
- 中文正文：MiSans、思源黑体或系统无衬线回退。
- 元数据与代码：等宽字体。
- 数字：必须使用 tabular 数字。
- 大数字遵循「越大越细」，不得使用 700-900 粗体制造廉价冲击。
- 1080×1440 卡片正文建议不小于 28px；元数据和来源不小于 20px。

### 4.3 固定识别件

所有版式必须包含：

1. 标的、代码、市场、模块和数据截至日。
2. 固定品牌名 `RODYA CAIJING STUDIO`。
3. 数据来源和口径。
4. 客户版固定免责声明或合规短句。
5. 同一套页边距、发丝线、字体和数字纪律。

## 五、信息优先级与留白纪律

### 5.1 信息优先级

卡片面积按以下顺序分配：

1. 核心结论。
2. 关键数字。
3. 历史、同业或同比环比对照坐标。
4. 风险、证伪条件和后续跟踪指标。
5. 来源、截至日和口径。
6. 图片。

图片不能挤占前五项。图片面积与文字发生冲突时，按「缩小图片、取消图片、拆页」顺序处理。

### 5.2 留白纪律

- 内容必须覆盖至少 75% 的 3:4 画布高度。
- 默认无信息留白控制在 10%-16%。
- 留白只能用于标题与数据、数据与结论、正文与页脚之间的层级停顿。
- 禁止为了高级感保留大块无信息区域。
- 禁止使用空白弹性容器把信息推到画布中段。
- 信息超量时拆页，不缩小正文和来源文字硬塞。

### 5.3 图片占比自适应

| 状态 | 图片占比 | 处理方式 |
| --- | ---: | --- |
| 信息适中且图片合格 | 38%-42% | 标准 B1 主视觉结论页 |
| 信息较多 | 25%-30% | 缩短图片区，保留结论和关键数字 |
| 信息密集或图片不合格 | 0% | 切换 B2、B3、B5 或 B6 纯数据构图 |

## 六、六种版式家族

### B1 主视觉结论页

用途：封面、重大财报、行业故事、阶段性核心结论。

结构：

- 上部高质量主视觉和一句结论。
- 下部 3-5 个关键数字、一个对照图、核心观察和验证点。
- 图片不承载事实，所有事实必须进入 HTML 数据区。

### B2 超级数字页

用途：估值分位、ROE、隐含增长率、破发率、孖展金额等单一主指标。

结构：

- 一个主数字。
- 至少一个历史或同业对照。
- 2-4 个支撑指标。
- 一个条件性结论或跟踪变量。

禁止只有裸数字和装饰性留白。

### B3 横向对比页

用途：公司对比、季度对比、情景对比、发行估值与同业对比。

结构：

- 两栏或多栏对照。
- 指标口径一致。
- 结尾解释差异来自哪里，不把排序直接写成买卖信号。

### B4 证据墙页

用途：财报原文、公告、招股书、截图、原始图表和监管文件。

结构：

- 一份主证据占较大面积。
- 1-3 份辅助证据交叉验证。
- 每份证据带来源、日期和一句解释。
- 截图文字必须在手机端可读，不可为凑数量缩成缩略图。

### B5 风险账本页

用途：排雷、证伪条件、风险跟踪、IPO 风险变量。

结构：

- 3-6 个风险行。
- 每行包含事实、当前状态、可观测指标和轻重级别。
- 末尾用「重点关注 X 项 + 常规监控 N 项」收口。
- 客户版不使用行动暗示型红绿灯。

### B6 产业链地图页

用途：产业链、利润池、瓶颈点、上下游与代表环节。

结构：

- 上中下游或输入、制造、应用三层关系。
- 利润池和瓶颈环节明确标出。
- 受益标的在客户版降级为相关公司或公开信息关联对象。

## 八、Claude 与 Codex 的能力分层

### 8.1 共同能力

- 共用一份结构化分析内核。
- 共用六种 HTML 版式和同一套 CSS 令牌。
- 共用客户版合规降级。
- 共用数据来源、截至日、确定性分级和免责声明。
- 必须能够在无图状态下完成高质量纯数据卡片。

### 8.2 Claude 路径

- 使用用户提供的照片、截图和公开材料。
- 没有合格图片时自动使用 B2、B3、B5 或 B6。
- 默认交付填好数据的 HTML，并提醒使用者浏览器截图。

### 8.3 Codex 路径

- 可调用生图工具生成无文字主视觉。
- 生图只能作为 B1 的背景或 B4 的非事实氛围素材。
- 标题、数字、图表、来源和免责声明全部由 HTML 排版。
- 默认仍交付 HTML；用户明确要求 PNG 时，使用环境现有截图能力渲染，不把渲染脚本加入 Skill。

## 九、主视觉质量门槛

### 9.1 合格方向

- 国际投行年报、机构研究报告和高端财经杂志气质。
- 建筑尺度、真实材质、机构研究桌面、工业与供应链真实场景。
- 石材、金属、玻璃、纸张、制造设备、实验室和真实产业空间。
- 构图必须为文字保留低细节安全区。

### 9.2 禁止方向

- 霓虹 K 线、发光数据流和赛博城市夜景。
- 西装人士握手、盯盘、指屏幕或假装开会。
- 金币、钞票、保险箱、豪车、名表等财富符号。
- 紫蓝渐变、玻璃拟态、廉价 SaaS 科技感。
- 假交易终端、假仪表盘、不可核实的 UI 和图表。
- 在生成图片中嵌入中文标题、数字、logo、水印或假标签。

### 9.3 图片验收

1. 主题与行业相关，而非泛财经装饰。
2. 存在足够的低细节文字安全区。
3. 光线克制，材质真实，无明显 AI 畸变。
4. 裁切后不损失主对象。
5. 缩略图状态下不抢夺核心数字。
6. 不合格即弃用，不降级凑数。

## 十、渲染流程

1. 子模块完成结构化分析内核。
2. 客户版按合规规则删除专业判断、目标价、买卖建议和动作暗示。
3. 根据卡片任务和字段数量选择 B1-B6。
4. 先排核心结论、关键数字、对照、风险和来源。
5. 判断剩余面积和图片质量，决定 41%、25%-30% 或无图。
6. 填充 HTML 模板。
7. 检查溢出、留白、字号、来源和手机缩略图。
8. 交付 HTML；按需提供 PNG。

## 十一、异常与降级

| 异常 | 处理 |
| --- | --- |
| 没有图片 | 使用纯数据版式，不提示失败 |
| 图片质量低 | 弃图并回退纯数据版式 |
| 数据字段过多 | 拆成两页，不缩字 |
| 缺少对照坐标 | 标为数据缺口，不用裸数字冒充结论 |
| 截图不可读 | 放大主证据或拆页 |
| Codex 生图不可用 | 与 Claude 路径一致，使用无图版式 |
| 文件写入或截图能力不可用 | 明确报告阻塞，保留完整 HTML |

## 十二、验收标准

### 结构

- 六种版式均能以 1080×1440、3:4 画布完整呈现。
- 内容高度覆盖率至少 75%。
- 无信息留白默认不超过 16%。
- 标题与下一信息块之间至少有明确视觉间距。
- 页脚不与正文碰撞。

### 可读性

- 手机缩略图中能识别标题、主数字和结论。
- 正文、来源和口径不低于字号底线。
- 图表和截图不因缩小失去意义。

### 财经纪律

- 关键数字包含值、对照、来源、截至日和口径。
- 红绿只表达涨跌方向。
- 不伪造数据、统计、截图或产品界面。
- 数据不足按共享降级规则标注，不生成强结论。

### 合规

- 客户版无目标价、买卖建议、收益承诺和操作指令。
- 风险内容使用事实语言和可观测指标。
- 港股打新自用方案不得泄漏到客户版。

### 跨模型

- Claude 可生成完整无图或用户素材版 HTML。
- Codex 可生成主视觉增强版 HTML。
- 两侧输出使用同一内核字段、版式名称和免责声明。
