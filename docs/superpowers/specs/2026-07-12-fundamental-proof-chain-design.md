# 基本面结论证明协议与框架外发现机制设计

日期：2026-07-12  
状态：待用户书面规格复核  
范围：`caijing-fundamental` 专业版；共享层只增加兼容字段，不改变其他模块默认行为

## 一、问题定义

当前基本面模块已经规定十个分析块、同业比较、杜邦、ROIC–WACC、单位经济和证伪链，但完成标准仍偏向“分析项是否出现”。模型可能用结果表格和结论摘要形式上覆盖章节，却省略口径调整、计算过程、比较坐标和结论如何成立的中间证据链。

同时，十个分析块和行业适配器不可能穷尽所有公司。特殊资产、交易结构、监管安排、会计口径、新业务、诉讼或资本动作可能成为真正决定公司质量的变量。固定框架必须保证最低覆盖，但不得限制新增研究问题。

## 二、目标与非目标

### 目标

1. 所有基本面专业版默认执行完整推导，不设置标准版与深研版双档。
2. 所有判断性结论均可反查到原始数据、口径、计算、比较坐标、反面证据和证伪条件。
3. 推导链断裂时停止对受影响事项下强弱、改善恶化或可持续性判断。
4. 十个分析块作为最低覆盖清单，同时强制执行框架外扫描与重要性判断。
5. 正文直接展示决定结论的关键推导，重复计算和底稿进入附录。
6. 保持行业适配能力，不把制造业公式强套到银行、平台、消费或研发型公司。

### 非目标

1. 本轮不建设覆盖所有行业的自动计算引擎。
2. 本轮不改变财报、估值、排雷、产业链和打新模块的默认输出行为。
3. 不以增加篇幅为目标，不允许重复扩写或弱证据填充。
4. 不要求每个事实描述都生成计算链；强制对象是判断性结论。

## 三、核心概念

### 判断性结论

包含“强、弱、改善、恶化、领先、落后、可持续、存在护城河、资本回报有效、成长质量高”等研究判断，或任何会进入首页研究结论、护城河评分、逻辑支柱及风险定性的表述。

纯粹转述已核实事实可以不计算，但必须带来源、口径和截至日。

### 结论证明链

每条判断性结论使用统一结构：

`研究问题 → 原始事实 → 口径调整 → 计算/拆解 → 历史坐标 → 同业坐标 → 研究判断 → 反面证据 → 证伪条件`

结构化字段如下：

```yaml
研究方法:
  结论证明:
    - 研究问题: ""
      关联分析块: ""
      原始数据:
        - 指标: ""
          数值: ""
          口径: ""
          截至日: ""
          来源: ""
          来源等级: "A|B|C"
      口径调整: "无需调整时写明原因"
      公式或拆解: ""
      计算结果: ""
      历史坐标: ""
      同业坐标: ""
      研究判断: ""
      反面证据: []
      证伪条件: []
      状态: "完整|断裂"
      断点: ""
      缺失数据: []
```

对其他模块该字段兼容可选；对 `caijing-fundamental` 专业版强制。

## 四、架构与文件职责

### `references/fundamental-proof-chain.md`

新增基本面专属执行事实源，负责：

- 判断性结论识别；
- 证明链字段、完成条件和断裂处理；
- 正文与附录分流；
- 框架外扫描、重要性 Gate 和行业适配器覆盖；
- 首页结论反向审查。

详细规则只保存在该文件，避免与 `SKILL.md` 重复。

### `caijing-fundamental/SKILL.md`

保留三问主线与十个分析块，新增以下强制流程：

- 执行前读取 `fundamental-proof-chain.md`；
- 十个分析块是最低覆盖，不是封闭分类；
- 每个进入最终研究判断的命题必须绑定完整证明链；
- 推导断裂时执行结论 Gate；
- 完成标准从章节出现升级为判断覆盖率和可复算性。

### `references/output-spec.md`

在共享内核中增加兼容字段 `研究方法.结论证明[]` 与 `研究方法.框架外发现[]`。明确只有基本面模块强制填充，其他模块不因本次升级改变行为。

### `references/sector-adapters.md`

增加证明链路由和覆盖规则：

- 制造业：量、价、产品结构、单位成本、产能利用率、资本回报；
- 消费与零售：销量/店数、客单价、同店、渠道、库存和费用效率；
- 软件与平台：用户、留存、ARPU、获客成本、服务成本和单位经济；
- 银行：规模、净息差、资产质量、信用成本、资本约束和股东回报；
- 保险：新业务价值、承保质量、投资收益、偿付能力与资本生成；
- 研发型公司：技术/临床里程碑、商业化概率、现金消耗与融资需求。

允许覆盖默认适配器，但必须记录默认指标失真原因、替代指标、结论连接方式和局限。

### `references/formulas-and-thresholds.md`

只补充稳定、可复算的公式和口径，例如杜邦、CAGR、现金覆盖、分位、利润桥恒等关系和基础单位经济。行业判断、会计调整和特殊交易结构保持研究判断，不脚本化。

### `references/docx-visual-spec.md`

新增推导展示纪律：

- 正文展示改变结论的计算、桥接表和比较坐标；
- 附录展示原始数据表、完整同业池、重复计算、口径调整和来源页码；
- 首页结论可点击或通过编号反查到对应证明链；
- 断裂链使用中性“未完成推导”状态，不用警报式视觉。

### 合同测试

新增基本面证明协议测试，验证所需文件包含强制规则与字段，并防止后续回退为结果表模式。

## 五、框架外发现机制

### 原则

十个分析块是地板，不是天花板。完成标准框架不能自动代表研究穷尽。

### 强制扫描问题

1. 是否存在与典型同业不同的资产、负债或收入结构？
2. 是否存在解释大部分利润但未被标准框架捕捉的变量？
3. 是否存在令常规指标失真的会计口径、交易结构或监管安排？
4. 是否存在足以改变原逻辑的新业务、重大诉讼、关联交易、技术路线或资本动作？
5. 是否出现财务数据与商业现实不匹配的异常？

### 重要性 Gate

框架外事项满足任一条件时进入正文并形成独立证明链：

- 可能改变收入、利润或现金流的主要来源；
- 可能改变资本结构、控制权或每股价值；
- 可能推翻护城河评分或投资逻辑支柱；
- 常规财务指标因它而失真；
- 公司、审计机构、监管机构或管理层讨论将其列为重大事项。

未通过重要性 Gate 的事项进入附录观察清单，不扩写正文。

### 结构化字段

```yaml
研究方法:
  框架外发现:
    - 事项: ""
      触发证据: ""
      标准框架未覆盖原因: ""
      重要性命中项: []
      处理: "新增正文分析块|附录观察"
      关联证明链: ""
      对原结论影响: ""
```

框架外事项不降低证据要求。资料不足时标记为“框架外未决事项”，暂停受影响结论。

## 六、运行流程

1. 识别公司类型、行业适配器、商业模式和默认关键指标。
2. 在套用标准框架前，先阅读年报、最新财报、审计意见、重大公告和管理层讨论，记录异常结构。
3. 以十个分析块建立基础研究问题，以框架外扫描补充公司特有问题。
4. 收集带来源等级、口径、截至日和页码的原始数据。
5. 对所有判断性命题建立证明链。
6. 执行结论 Gate：完整链才允许定性；断裂链只保留事实、断点和补数要求。
7. 生成结构化内核，再将关键推导渲染进正文、底稿渲染进附录。
8. 从首页逐句反查证明链；无法反查的判断删除或降级。

## 七、错误与资料不足处理

### 推导链断裂

断裂链必须同时输出：

- 已核实事实；
- 推导停止位置；
- 缺失数据；
- 受影响的分析块和首页结论；
- 可以继续观察的替代事实指标。

不得输出条件性强弱判断，不得用权威机构预测值替代缺失的公司事实完成链条。外部预测可以作为单独的预期坐标，但不能伪装成公司实际数据。

### 历史或同业坐标不适用

必须说明不适用原因和替代坐标。仅写“缺数据”不能自动豁免；需要列出已尝试的数据源和补数要求。

### 证据冲突

公司披露、监管文件与权威二手数据冲突时，分栏展示口径并暂停相关定性，直到差异可以解释。

## 八、正文与附录

### 正文必须展示

- 决定首页研究判断的证明链摘要；
- 利润来源或经营驱动桥；
- 关键历史和同业坐标；
- 护城河评分的逐分证据与降分条件；
- 框架外重大事项；
- 支持证据、反面证据和证伪条件。

### 附录必须展示

- 原始数据和来源页码；
- 口径调整表；
- 完整公式与重复计算；
- 完整同业池及纳入/排除理由；
- 未通过重要性 Gate 的框架外观察项；
- 断裂链与补数清单。

## 九、完成门禁

基本面专业版只有同时满足以下条件才能标记完成：

1. 所有判断性结论的证明覆盖率为 100%。
2. 强判断不存在断裂链。
3. 关键计算能由报告中的数据、口径和公式复算。
4. 历史与同业坐标均提供；不适用时有原因和替代坐标。
5. 每条核心逻辑均有支持证据、反面证据和证伪条件。
6. 框架外扫描已执行并记录。
7. 行业适配器的采用或覆盖理由已披露。
8. 首页、正文与附录使用同一份结构化内核。

## 十、验证设计

### 合同测试

至少覆盖以下失败情形：

- 只有结果表，没有 `结论证明` 字段；
- 推导状态为断裂，却仍保留强判断；
- 关键判断只有历史比较，没有同业坐标且未解释；
- 十块完成后未执行框架外扫描；
- 覆盖行业适配器但未说明理由；
- DOCX 规范未区分正文关键推导与附录底稿。

### 回归场景

1. 常规模型公司：验证十块覆盖与证明链完整性。
2. 行业特殊公司：验证不会强套制造业指标。
3. 框架外事项明显的公司：验证能够主动新增研究问题。

### 真实样张验收

宁德时代 V2 作为首份真实验收件，至少新增：

- 利润来源桥；
- 量价成本或可得替代经营桥；
- 同业比较坐标；
- 护城河逐分推导；
- 正文关键推导与附录底稿；
- 框架外扫描结果。

V2 仍放在样张审核区，用户明确批准后才替换公开样张。
