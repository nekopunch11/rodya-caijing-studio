# 财经·排雷

## 作用

`caijing-risk` 用来回答“有没有雷”。它聚焦财务、治理、经营和交易结构风险，用公告、年报附注、监管记录和模型预警做公开信息扫描。

## 适合场景

- 客户问“这家公司安全吗”“有没有雷”。
- 需要核查审计意见、质押、资金占用、违规担保、诉讼、监管记录等风险。
- 港股标的需要额外查看合股、供股、CCASS 集中度、停牌史等信号。
- 想提前扫描收入放缓、毛利率压缩、应收存货快于收入、现金流弱于利润等恶化因子。

## 核心优点

- 学术模型和本土红旗库结合：Beneish M-Score、Altman Z-Score、A 股红旗、港股红旗。
- 引入跨市场恶化因子扫描，但只作为风险提示线索，不写成做空依据。
- 一票否决项优先检查，重大风险不会被加权平均掩盖。
- 每面红旗必须有公告或报表证据，没有证据不挂旗。
- 客户版已降级为公开信息提示项，降低“评级即建议”的误读风险。
- 增加模型原始适用范围、本土局限与误报解释；每个模型信号回到附注/审计/监管证据验证，审计事项拆成意见、金额、持续期、整改和实际影响。

## 合规边界

- 不使用“造假、骗子、必爆、庞氏”等定性或煽动词。
- 模型结果只能写“进入统计预警区间”，不能写成事实定罪。
- 客户版只列事实、提示项数量、重点关注项和免责声明，不做爆雷预测。

## 默认产出

- 专业版：排雷报告，含模型体检与误报控制、审计事项桥、红旗逐项扫描、恶化因子扫描、四类雷汇总、一票否决核查、证据清单。
- 客户版（按需，默认只出专业版，用户要转客户时再出）：风险提示卡结构稿 + 150-300 字转发文案。

## 执行事实源

实际执行规则以 `caijing-risk/SKILL.md` 为准；模型公式、阈值、取数降级和客户版降级以 `references/` 中对应文件为准。
