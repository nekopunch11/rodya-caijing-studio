# 财经·产业链

## 作用

`caijing-industry` 用来回答“题材链条怎么分、代表公司有哪些”。它是题材层模块，不评价单个公司的质地、估值或风险，而是拆链条、定义行业关键指标、看利润池、分类竞争方式、列代表公司。

## 适合场景

- 热点题材起来后，需要快速梳理上中下游。
- 想区分真实业务相关公司和低相关蹭题材公司。
- 需要把题材从“概念”拆成环节、价值量、竞争格局和纯度。
- AI、半导体、机器人、新能源、先进制造等题材，需要判断是否存在供应链瓶颈卡位。

## 核心优点

- 用渗透率 S 曲线判断题材处于 0 到 1 主题期，还是 1 到 N 兑现期。
- 利润池升级为五维证据评分：价值量、毛利率/单位利润、集中度、议价权、供给约束；不把不同量纲直接相乘成伪精确结果。
- 按平台型、制造型、渠道型、国产替代、全球竞争等方式分类竞争格局。
- 对适用题材启用瓶颈点方法，并建立后续验证链。
- 代表公司拆为业务纯度、增量收入弹性、增量利润弹性、竞争优势与估值已反映程度，避免把纯度排行当作受益或投资排行。
- 客户版使用“代表公司示例”，不做推荐排序。

## 合规边界

- 标的清单是客观图谱，不是股票池。
- 不使用“首选”“最受益”“受益最大”等推荐感表达。
- 纯度缺少披露时标“未披露，待核实”，不得编数。

## 默认产出

- 专业版：产业链梳理报告，含题材驱动、行业关键指标、生命周期、图谱、利润池五维评分、竞争分类、代表公司五层暴露表、瓶颈点验证链（如适用）。
- 客户版（按需，默认只出专业版，用户要转客户时再出）：图谱卡结构稿 + 150-300 字转发文案。

## 执行事实源

实际执行规则以 `caijing-industry/SKILL.md` 为准；取数降级和客户版合规降级以 `references/` 中对应文件为准。
