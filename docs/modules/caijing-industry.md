# 财经·产业链

## 作用

`caijing-industry` 用来回答“题材链条怎么分、代表公司有哪些”。它是题材层模块，不评价单个公司的质地、估值或风险，而是拆链条、看利润池、列代表公司。

## 适合场景

- 热点题材起来后，需要快速梳理上中下游。
- 想区分真实业务相关公司和低相关蹭题材公司。
- 需要把题材从“概念”拆成环节、价值量、竞争格局和纯度。

## 核心优点

- 用渗透率 S 曲线判断题材处于 0 到 1 主题期，还是 1 到 N 兑现期。
- 用价值量占比和 CR3/CR5 判断利润可能沉淀在哪个环节。
- 用相关业务收入占比做“纯度”过滤，避免只按概念标签列公司。
- 客户版使用“代表公司示例”，不做推荐排序。

## 合规边界

- 标的清单是客观图谱，不是股票池。
- 不使用“首选”“最受益”“受益最大”等推荐感表达。
- 纯度缺少披露时标“未披露，待核实”，不得编数。

## 默认产出

- 专业版：产业链梳理报告，含题材驱动、生命周期、图谱、利润池、代表公司纯度表。
- 客户版：图谱卡结构稿 + 150-300 字转发文案。

## 执行事实源

实际执行规则以 `caijing-industry/SKILL.md` 为准；取数降级和客户版合规降级以 `references/` 中对应文件为准。
