# 时效核验 Gate（freshness-gate）

任何模块产内核前，先过本 gate：确认输入材料和统计基准仍然新鲜。**过期数据必须提示时效风险，不硬出强结论。**本文件与 `data-fallback.md` 衔接——gate 判定"超窗"后，按 data-fallback 标注数据档次、结论边界和补充要求。

## 一、通用规则

1. 每次执行开头，先确认**当前日期**（用运行环境时间，不凭训练记忆）。
2. 所有关键输入必须带"数据日期"；与当前日期做差，对照下方窗口表。
3. 超窗且可联网 → **必须重取最新数据**后再分析，不得直接引用旧值。
4. 超窗且无法核实（联网失败/无来源）→ 结论区显式标注"基于 X 日前数据，时效存疑"，确定性降一级；客户版不出档位/评级，专业版保留框架但把受影响结论写成条件性判断。
5. skill 文件内置的静态数据（券商费率表、案例库、统计基准、机制描述）都带"截至日期"——超窗时视为**线索而非事实**，核实后才能进入计算。

## 二、时效窗口表

| 数据类型 | 窗口 | 超窗动作 | 主要涉及模块 |
|---|---|---|---|
| 财报期 | 必须为公司最新披露期 | 见财报 gate | earnings |
| 招股期状态 | 认购截止时间之前 | 见招股期 gate | ipo-a / ipo-hk |
| 孖展/申购热度数据 | ≤24 小时（招股期内逐日变化） | 重取；不可得则标截至时点，热度因子降为"仅供参考" | ipo-hk |
| 破发率/情绪窗口统计 | ≤3 个月 | 重新拉取近月统计再分析 | ipo-a / ipo-hk |
| 券商费率/杠杆预设 | ≤3 个月 | 标"以券商 App 为准"；涉及金额计算须先核实 | ipo-hk |
| 一致预期/估值分位基准 | ≤1 个月 | 重取；不可得则标注数据日期 | earnings / valuation |
| 市场机制/制度规则 | 内置描述超 6 个月 | 搜索核实有无新规再引用 | ipo-a / ipo-hk |
| 行情/股价类引用 | 当日，或显式标明日期 | 标明数据日期，不做时点模糊表述 | 全部 |

## 三、模块特定 Gate

### 财报 gate（earnings）

- 分析前核实所评财报是否为**最新披露期**：搜 `{公司} 最新 财报/业绩公告`，对照披露日历。
- 已有更新一期 → 提示用户并默认改评最新期；用户坚持评旧期 → 全文显式标注"非最新期，历史复盘口径"。
- 同时核对是否存在业绩预告/快报晚于正式财报的增量信息，有则并入市场视角块。

### 招股期 gate（ipo-a / ipo-hk）

- 核对**认购截止时间 vs 当前时间**，三种状态三种口径：
  - **招股期内** → 正常流程；ipo-hk 孖展数据以小时计时效，必须标截至时点。
  - **已截止、未上市** → 切换"待上市跟踪"口径：不出申购方案，跟踪定价结果、暗盘安排。
  - **已上市** → 切换"复盘模式"：对照实际暗盘/首日表现检验判断，不再输出任何申购相关计算。
- ipo-hk 自用模式额外核验：券商费率预设、破发率统计基准、机制描述是否超窗（见窗口表），超窗先核实再算钱。
- ipo-hk 识别 18C 时额外核验其指定公开发售分配机制与最新豁免；不得默认其可按普通发行人自由采用机制 B。

## 四、执行位置

- 父 skill 通用工作流**第 2 步（取数）内嵌本 gate**，所有模块生效。
- `caijing-earnings`、`caijing-ipo-a`、`caijing-ipo-hk` 在各自工作流第 1 步**显式执行**对应模块 gate。
