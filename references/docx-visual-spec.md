# docx 专业版视觉规范（唯一视觉事实源）

> 定位：**本土券商研报骨架 + 国际卖方图表纪律 + 炭黑金克制配色**。适用于所有 `caijing-*` 单模块专业版 `.docx`。改视觉只改本文件，七模块全生效。
> 内容/结构（章节大纲、分析块）仍以各模块 SKILL.md 与 output-spec.md 为准；本文件只管"长什么样"。

## 一、气质与配色

- 整体：暖白纸底、炭黑主色、古铜金点睛。**单色纪律**——除炭黑+金外不引入第二品牌色。密而有层级，不追求大留白（受众是专业投顾）——与卡片 / PPT 统一走「数据台密度」（见 card-components.md §一）。
- 色值：

| 用途 | 色值 |
|---|---|
| 纸底 | 暖白 `#FDFDFB` |
| 主色炭黑 | `#1A1A1A`（机构色带、表头、标题、正文） |
| 点睛古铜金 | `#A8894F`（细线、竖条、角标、专业判断标签底） |
| 金字（深底上） | `#C8A55E`；金字（浅底上） | `#8A6D1F` |
| 正文文字 / 次级 / 细线 | `#1A1A1A` / `#6B7280` / `#E6E6E6` |
| 斑马纹隔行 | `#F6F5F1` |
| 涨（红） / 跌（绿） | `#C0392B` / `#1E9E5A` |
| 估算标记 | 浅金底 `#F0E6CE` + 金字 `#8A6D1F` |

- **红涨绿跌锁 A 股/港股惯例**：红=上涨/超预期、绿=下跌/不及预期，**只染数字与箭头**，绝不用于大色块或标题，任何图表不得反用。
- 确定性分级 A/B/C：字母用**金色加粗**呈现于首页问责块与各分析块标题旁，不另做红绿灯。

## 二、字体（免费可商用可嵌入优先）

| 层级 | 字体 | 回退 | 字号/字重 |
|---|---|---|---|
| 报告标题 | MiSans / 思源黑体 | 微软雅黑 → 黑体 | 19pt 粗 |
| 章节标题 H1（一、二、三） | MiSans / 思源黑体 | 微软雅黑 | 13pt 粗 |
| 子目 H2（1. 2. 3.） | MiSans / 思源黑体 | 微软雅黑 | 11.5pt 粗 |
| 正文 | 思源宋体 | 宋体 / SimSun | 10.5pt，行距 1.9，两端对齐，首行缩进 2 字 |
| 表格文字 | 表头 MiSans / 正文思源宋体 | 黑体 / 宋体 | 10.5pt |
| 脚注·来源 | 思源宋体 | 宋体 | 9pt 灰 |
| 机构色带·角标 | MiSans | 黑体 | 9–10pt |

- **数字统一用 tabular figures（等宽）**：思源宋体 tabular 或 Arial，关键数字表与正文数字一律对齐。
- **版权纪律**：正文思源宋体（Source Han Serif，SIL OFL）、标题 MiSans（小米免费商用），二者均可**嵌入 docx / 导 PDF / 转图**，用于对外分发。SimSun/微软雅黑仅作系统回退，**不可嵌入或转图分发**。分发前若嵌入字体，只嵌思源宋体 + MiSans。

## 三、页面

- A4 纵向，页边距 上下 2.5cm / 左右 2.8cm，单栏。

## 四、首页版式（自上而下）

1. **机构色带**：满宽炭黑底 `#1A1A1A`。左「财经内容台」金字（MiSans，字距 3px）；右「公司深度 · {模块名}」浅金字。
2. **标题行**：左黑体报告标题「{标的}（{代码}）{模块}研报」；右**问责块**（右对齐，MiSans 9pt 灰，三行）：
   - `数据截至 YYYY-MM-DD`
   - `确定性 {A/B/C}`（字母金色粗）` · {市场}`
   - `制作 {skill 名，如 rodya-caijing-studio 或子模块名}`
   - 下 0.5pt 灰线收口。
3. **投资要点框**：黑体「投资要点」标题带 4px 金色左竖条；宋体正文含 事实摘要 →〔研究判断〕（金字括注）→ 判断依据 → 关键变量。客户合规版保留研究判断，但不显示行动建议。
4. **关键数字表**（见六）。
5. **图 1 …**（见七）。
6. 正文起「一、公司速览」。

## 五、页眉页脚（第 2 页起）

- 页眉：左「{模块名} · {标的}{代码}」，右「数据截至 YYYY-MM-DD」，下金色细线。
- 页脚：左 slogan「客观体检 · 不含买卖建议」，右「第 X / N 页」，上灰色细线。

## 六、关键数字表

- **数字四件套（全文通用硬要求）**：任何关键数字（表内与正文内）都必须带四件套——数值 + 对照坐标（同业分位 / 历史分位 / 同比环比 之一）+ 来源 + 截至日与口径。**裸数字（无对照、无来源）视为未完成**，这是"内行"与"外行"最直接的分野。
- 表头炭黑底 `#1A1A1A` 白字（MiSans）；列：指标 / 数值 / 对比 / 状态（首页版可省"来源"列，改用表下来源行）。
- 数值列右对齐 tabular，按涨跌染色（涨红/跌绿），中性指标用炭黑。
- 隔行斑马 `#F6F5F1`。
- 状态列：`已核实`（灰）/ `估算`（浅金底 `#F0E6CE` + 金字）。
- **表下必附来源行**（9pt 灰）：`来源：{数据源}，截至 YYYY-MM-DD。`

## 七、编号图表（国际卖方纪律）

- 每张图/表编号并起标题在上：`图 1`、`图 2`、`表 1`（黑体 11pt）。
- **每张图/表下必附来源行**（9pt 灰，来源 + 截至日）；**无来源不出图**。
- 风格：细线、无 3D、无阴影，只用炭黑 + 金 + 涨跌色，与正文一致。
- 常用图型：
  - **估值分位 band**：横条（底 `#E6E1D6`，金填充至当前位），当前点炭黑圆点，两端「低/高」，下附「当前 X% 分位」。
  - **护城河五源**：■□ 方块显示 0–2 分/源（满格 2 ■）。
  - **杜邦三因子**：小条形或表格。
  - **三情景 / 风险回报竖条**（bear/base/bull，借大摩式竖轴）：**仅专业版**，客户版丢弃。

## 八、测算透明（假设框）

- 每个涉及假设的测算（WACC / ROIC 口径、反向 DCF 隐含增长、情景假设、可持续增长率等）旁必配**浅灰假设框**，列：关键假设、取值、敏感度（±X 假设变动对结果的影响）。
- 外行给结果、内行给"结果 + 假设 + 敏感度"；**无假设披露的测算视为未完成**，不得只甩一个数字。

## 九、结论推导与计算底稿（证明链沉附录，正文只见叙事）

- **正文是研报叙事，不是证明链清单**：正文用自然语言把判断与支撑它的关键数字讲成连贯论证，判断句末用证明链编号回指（如 `[P-03]`）。**正文不得出现证明链结构词或字段表**——链状态、嵌入证明链、原始数据与口径、口径调整、计算与比较坐标、拆解方法、关键推导、未完成推导、证据锚 等一律不进正文，详细规则见 `fundamental-proof-chain.md` §八。
- **完整证明链集中进附录**：以台账形式展示原始数据、来源页码、完整同业池及纳入/排除理由、口径调整表、公式/拆解、重复计算、历史与同业坐标、断裂链和补数清单。正文、首页、附录共用同一编号，保证反向复核。
- **不设自述方法论章节**：报告不出现"分析框架与证明链地图""证明链状态表""审核规则"等解释报告自身构造的页面（见 `fundamental-proof-chain.md` §八）。
- 断裂链在正文只用一句中性叙述说明"暂不定性 + 需跟踪项"，不做红色警报视觉，也不得在首页保留对应定性结论；完整断点与缺失数据进附录。
- **章节标题只用中文「一、二、三」编号，不加英文 kicker 或双语代码**（如 `01 EXECUTIVE SUMMARY`、`FUNDAMENTAL FRAMEWORK`）——重复的英文标签是噪音，削弱研报专业感。首页机构色带的英文角标不受此限。

## 十、风险章节渲染要求（专业感核心）

风险的**完备**是最强专业信号，但呈现不当会显悲观、也显业余。以下为硬性要求（`caijing-risk` 全量、其余模块的风险块同样适用）：

1. **分维成表，不流水账**：风险按维度成体系呈现（经营 / 财务 / 治理 / 政策 / 估值 / 流动性 等适用维度），首列维度、其后风险点——让读者看到"每个维度都扫过了"。完备感来自维度覆盖，不是数量堆砌。
2. **每条风险 → 可观测指标**：每个风险点必须落到一个可跟踪的量（如"商誉/净资产 32%，跟踪年报减值测试"）并写触发信号与影响路径（若…则…）。**禁止只写"某某风险"而无观测抓手**——这也是去悲观最有效的一招（风险从"悬顶之剑"变成"仪表盘读数"）。
3. **标当前状态与是否已反映**：每条标注 已发生事实 / 潜在 / 尾部，并注明是否为市场已知（是否已被 price in）。
4. **缓释因子对称呈现**：每条风险旁列缓释 / 对冲因素（如有）；风险与支撑并列，不单列风险，防悲观感。
5. **事实语言，禁定性煽动词**：只陈述公告与报表事实并带来源，禁"造假 / 暴雷 / 崩盘 / 必爆"等词（同 `caijing-risk` 措辞红线）；M/Z 模型结果表述为"进入统计预警区间"，非"在操纵利润"。
6. **体检框架，非看空框架**：以"客观体检查异常"呈现，不作看空结论；表头 / 引语用中性词（如"公开信息提示项"）。
7. **轻重收口**：风险表后必给收口——重点关注 1–2 项 + 常规监控 N 项；不留无权重的长清单（无权重令人焦虑，给了轻重反而踏实）。专业版可用总评灯（仅专业版渲染），客户版用"提示项计数 + 重点 X 项"替代。

**证伪条件与跟踪清单置于显眼位**：紧接风险表或投资逻辑支柱后，用独立小框 / 小节列——"哪些公开信息变化会推翻核心判断" + "后续跟踪的财报 / 订单 / 价格 / 政策节点"。这是把风险转成前瞻观察、去悲观化的关键件。

## 十一、章节与正文

- 章节标题黑体带「一、二、三…」编号（H1 13pt）；子目「1. 2. 3.」（H2 11.5pt）。
- 正文思源宋体 10.5pt、行距 1.9、两端对齐、首行缩进 2 字。
- **禁止为塞进 N 页压字号或压行距**：正文字号固定 ≥10.5pt、表格 ≥9pt；报告篇幅是充分论证（研究型段落、非结论式一句）的自然结果，**不预设目标页数**。宁可多页，不可为省页把字压小、把分析压成一句。
- 来源用**脚注制**：关键数字上标序号，章节末或页脚列来源（优于行内括注，正文更干净）。

## 十二、文末

- 数据附录：来源清单表 + 确定性分级说明 + 数据缺口声明。
- 固定免责声明段：思源宋体 9pt 灰，浅灰底框（文案见 compliance.md 第四节）。

## 十三、口径切换（客户版渲染）

- 客户版 docx 保留〔研究判断〕、判断依据、适用边界、反面证据和情景条件；去除行动建议、个性化资金、目标价和收益导向组件。其余版式沿用本规范。

## 十四、生成实现

- 执行链分为两条：其他执行环境的兼容链由 `scripts/docx_renderer.py` 从 `assets/docx-template.docx` 起稿并读取 `references/docx-style-tokens.json`；**当客户使用 Codex 生成 DOCX 时，默认直接执行本视觉规范，由 `scripts/docx_spec_renderer.py` 生成 Word 原生段落、表格和 OOXML 样式，不调用通用 `docx_renderer.py`**。各模块只提供结构化内容块；禁止模块自行创建 `Document()` 或硬编码另一套颜色/字号/页边距。
- 每次生成后先运行 `scripts/audit_docx_visual.py`，再按文档 skill 执行 render → PNG → 逐页检查；审计失败或渲染未完成均不得声称视觉规范通过。
- 用 docx skill / python-docx 生成，字体走上表回退链；对外分发需嵌入时只嵌思源宋体 + MiSans。
- 本文件是 docx 视觉唯一事实源；卡片视觉见 card-components.md，PPT 视觉见 output-spec.md §三③及 assets/deck-template.mjs。
