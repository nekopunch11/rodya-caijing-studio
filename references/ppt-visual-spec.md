# PPT 视觉规范（唯一执行事实源）

本文件是财经内容台 PPT 的唯一执行视觉事实源。`docs/` 下的设计文档仅供人类阅读，不参与运行时执行。

## 一、定位与触发

- PPT 仅在用户明确点名“汇报 / PPT / deck / 幻灯”时生成，多模块分析不自动生成。
- 默认受众：**投顾内部研究汇报**。
- 默认文件：可编辑真 `.pptx`；用户只要浏览器演示或 PDF 时可用 HTML deck。
- 本轮不定义面向客户的路演版；需要客户口径时只做合规降级，不自动套另一套营销视觉。
- 高级感来自排版、字体、留白、密度、版式与图片比例，不来自固定行业配色或单一视觉隐喻。

## 二、页面叙事

先从结构化分析内核提炼：

1. 研究命题；
2. 核心矛盾；
3. 支撑命题的主指标；
4. 对照坐标；
5. 风险与证伪条件；
6. 数据来源与截至日。

每页只承担一个叙事任务，标题优先写成可直接说出口的研究判断，不用“综合体检”“基本面分析”等泛化报告名占据最大层级。

## 三、共用数据契约

真 `.pptx` 与 HTML deck 必须消费同一份数据，不得分别撰写分析。

```yaml
deck:
  title: 研究命题
  subject: 标的 / 代码 / 市场
  as_of: YYYY-MM-DD
  theme:
    mode: dark | light | media-led
    accent: 可选六位十六进制颜色
  slides:
    - type: cover | overview | thesis | fundamental | valuation | risk | disclaimer | pipeline
      conclusion: 本页结论
      eyebrow: 可选眉题
      hero_metric: {label: "", value: "", context: "", direction: up | down | neutral}
      support_metrics:
        - {label: "", value: "", context: "", direction: up | down | neutral}
      evidence:
        - {label: "", value: "", source: "", as_of: ""}
      watchlist:
        - {risk: "", status: "", mitigant: "", indicator: ""}
      visual_asset:
        path: 可选本地图片路径
        alt: 图片替代说明
        position: left | right | full
```

最低校验：

- `deck.title`、`deck.subject`、`deck.as_of` 必须存在；
- 每页必须有 `type` 和 `conclusion`；
- `visual_asset.path` 不存在时删除该字段并切换无图布局；
- 关键数字必须带对照坐标、来源和截至日；
- 对客口径按 `compliance-rendering.md` 降级，不修改内核。

## 四、画布、网格与密度

- 画布：16:9。
- 基础网格：十二列。
- 左右安全边距：画布宽度 4%–6%。
- 顶部功能区：画布高度 7%–12%。
- 底部来源区：画布高度 4%–7%。
- 内容页有效内容占用：70%–85%。
- 单块无信息留白通常控制在 10%–16%。
- 留白用于层级和阅读节奏，不制造海报式空洞。
- 内容超量时依次：删低价值文字 → 缩减或取消图片 → 拆页；不得缩小到不可读字号。
- 直角为默认，圆角为 0；发丝线只用于分组、对齐和坐标。
- 禁止以等宽圆角卡片、胶囊、阴影浮层作为默认容器。

## 五、字体系统

| 角色 | 首选 | 回退 |
| --- | --- | --- |
| 中文标题 / 正文 | MiSans / MiSans VF | Microsoft YaHei / PingFang SC / sans-serif |
| 英文 / 数字 / 股票代码 | IBM Plex Mono | JetBrains Mono / Consolas / Courier New |

字号建议：

- 封面研究命题：40–54pt；
- 页面标题：30–36pt；
- 正文与结论：16–20pt；
- 主数字：40–72pt；
- 辅助数字：22–34pt；
- 标签、来源、页码：9–12pt。

数字使用 tabular 对齐。标题通过字重、位置和对比建立层级，不依赖极端大字。中文正文不用等宽字体。

## 六、自适应主题令牌

所有颜色只能从以下功能令牌读取：

```yaml
surface: 页面底色
surface_alt: 次级底色
ink: 主要文字
muted: 辅助文字
hairline: 发丝线
accent: deck 唯一点睛色
up: 涨 / 正向变化
down: 跌 / 负向变化
scrim: 图片文字遮罩
```

三种模式：

- `light`：高明度中性底，适合大多数研究汇报；
- `dark`：低明度底，适合暗色资产或需要更强对比的 deck；
- `media-led`：从主视觉或品牌资产取得一个低饱和度 `accent`，其余令牌保持中性。

`accent` 选择顺序：用户指定品牌色 → 用户品牌资产 → 主视觉低饱和度颜色 → 行业主题色 → 中性点睛色。

颜色纪律：

- 一套 deck 只有一个 `accent`，占画面比例约 5% 以内；
- `up/down` 只用于有方向含义的数据，不能作为装饰；
- 图片可保留自身颜色，但需通过裁切、遮罩、降低饱和度或统一明度融入主题；
- deck 内主题锁定，不逐页更换颜色语言；
- 不固定冷银、铜金、蓝色、行业色或墨账本配色。

## 七、研究命题与微标签

封面层级：标的 / 代码 → 研究命题 → 分析范围 / 数据截至日 → 核心判断。

- “内部研究”全页最多出现一次；
- 不同时堆叠英文 badge、中文口径、版本号和状态标签；
- 元数据用列对齐或发丝线分组；
- 页码和来源是功能件，保持小层级；
- 可使用“核心矛盾、逻辑支柱、估值消化、证伪条件、跟踪重点”等内部研究语言；
- 禁止营销词、收益承诺和无证据的确定性语气。

## 八、`1+3` 数据层级

- `1`：一个最能支撑本页结论的主指标；
- `3`：最多三个辅助指标，提供对照、质量或风险解释。

每个数字必须包含：值 + 对照坐标 + 来源 + 截至日 / 口径。

总览表、风险清单、Pipeline、同业比较表和免责声明不强制 `1+3`，但仍必须有明确结论。图表、分位、估值 band 和比较关系用原生 PPT 图表、形状或表格绘制；AI 生图不得生成数字或图表。

## 九、七类固定版式

### 1. `cover` 研究命题封面

- 研究命题为最大标题；
- 一句核心判断；
- 有图时占画面 35%–50%；
- 无图时改为非对称纯排版，不保留图片槽。

### 2. `overview` 综合结论总览

- 汇总模块结论、主数字、确定性和数据缺口；
- 使用账本表或非对称分组；
- 默认无图。

### 3. `thesis` 投资逻辑与核心矛盾

- 一个研究命题；
- 2–4 个逻辑支柱；
- 每个支柱带可观测指标和证伪条件；
- 图片可选，占比不超过 30%。

### 4. `fundamental` 基本面 / 财报模块

- 默认 `1+3` 数据层级；
- 一段专业判断和跟踪变量；
- 图片为 0%–25%。

### 5. `valuation` 估值与同业坐标

- 主数字为估值分位、隐含增速或关键坐标；
- 配合区间、同业或历史坐标；
- 默认无装饰图片。

### 6. `risk` 风险、证伪与跟踪

- 风险 → 当前状态 → 缓释因子 → 可观测指标；
- 结论分轻重；
- 默认无图。

### 7. `disclaimer` 免责声明

- 免责声明、数据截至日、来源口径完整；
- 继承封面主题，不引入新题材。

### 可选 `pipeline`

产业链项目可插入上游 / 中游 / 下游、利润池、瓶颈点和代表公司关系页。它不是固定第几页，也不计入七类固定页。

## 十、内容驱动图片系统

图片不是默认必需件。只有能改善叙事、行业理解或视觉节奏时才使用。数据密度高、图片无新增信息、图片质量不足、图片挤压结论或可能造成公司事实误认时，直接使用无图版式。

图片比例：

| 页面 | 图片占比 |
| --- | --- |
| 封面 | 35%–50% |
| 章节 / 过渡 | 40%–60% |
| 公司 / 产业链内容页 | 25%–40% |
| 基本面 / 财报页 | 0%–25% |
| 估值 / 风险页 | 0%–20%，默认无图 |

信息优先级：研究命题 > 核心判断 > 主数字 > 对照坐标 > 风险 / 证伪 > 来源 / 截至日 > 图片。

生图变量：`domain`、`thesis`、`role`、`placement`、`negative_space`、`theme_mode`、`accent_hint`、`realism`。

生图硬限制：

- 不生成文字、数字、Logo、股票代码、图表、金融结论或水印；
- 不虚构公司厂房、产品外观、管理层、门店或办公地点；
- 无法核实具体场景时只生成行业主题视觉，不伪装成公司实景；
- 禁止握手、金币、上涨箭头、K 线屏幕、城市股票屏等廉价财经隐喻；
- 禁止默认赛博霓虹、实验室、机房或完美 CGI；
- 使用真实材质、自然光、轻微使用痕迹和编辑摄影质感；
- 每套 deck 只使用一个图片风格家族。

## 十一、Codex / Claude 双能力路径

两端共用结构化内核、研究命题、页面规划、主题令牌、版式、数据、来源和输出契约。

- Codex：图片决策为“需要”且 imagegen 可用时，先生成本地图片资产，再把路径交给 renderer；图片不通过质量检查时弃图。
- Claude：使用用户提供图片，或直接使用高级纯排版与原生图表；不得保留空图片框、低质量占位图或失败提示。
- renderer 不直接依赖 imagegen API，只接收可选本地路径。

## 十二、失败处理

| 情况 | 处理 |
| --- | --- |
| imagegen 不可用 | 使用无图或用户供图路径 |
| 图片质量不足 | 弃图并重新排版 |
| 图片可能与公司事实混淆 | 改为行业抽象视觉或无图 |
| 字体缺失 | 使用回退链，不阻塞生成 |
| 内容过多 | 删低价值内容、取消图片或拆页 |
| 页面溢出 | 调整结构或拆页，不压缩到不可读字号 |
| PPTX 与 HTML 不一致 | 以结构化内核和本文件为准同时修复 |

## 十三、交付前 QA

1. 七类固定页均可生成；产业链另测 Pipeline。
2. 同一数据分别生成有图与无图样张，差异只能是 `visual_asset` 和空间分配。
3. 每页单独渲染 PNG 并逐页检查；contact sheet 只看整体节奏。
4. 使用 `slides_test.py` 检查越界。
5. 检查图片裁切、遮罩、字体回退、换行、来源和页码。
6. 使用 Microsoft PowerPoint 原生导出至少复核封面、估值和风险页。
7. 一套 deck 只有一个 `accent`；`up/down` 只用于方向数据。
8. 每页“内部研究”最多一次，不存在空图片槽或未替换占位符。
9. renderer 不包含样例公司、样例数字或固定图片路径。
10. 源目录与唯一 Codex 运行副本的关键执行文件 SHA-256 一致。
