# 统一专业响应层（内核 schema + 受众口径 + 交付形态）

## 一、地基原则：一份内核、三形态渲染

每次分析只产出**一份「结构化分析内核」**；docx、卡片、PPT 都是这份内核的不同渲染视图。**绝不允许**为不同形态分别撰写分析——那会导致三处不一致、改一处漏两处。修改分析时改内核，再重新渲染。

## 零、交付硬门槛（先执行）

- 用户触发任一 `caijing-*` 模块且未明确要求“只在聊天里回答”时，**专业版默认必须生成文件成品**：单模块生成 `.docx`；多模块组合默认逐模块生成 `.docx`（或合并摘要），仅当用户点名“汇报/PPT/deck/幻灯”时才生成 PPT。聊天区只能交付摘要、文件路径、数据截至日、确定性分级、数据缺口和未完成项。
- **客户合规版默认不出、按需生成**：单模块默认只交付专业研究 `.docx`。用户说“转客户/客户版”但未指定形态时默认生成卡片+文案；说“客户报告”时生成客户合规 docx；说“客户 PPT/客户汇报”时生成客户合规 `.pptx`。客户版逐句过 compliance-rendering.md，但保留证据化研究判断。
- **不得用几段 Markdown 短评替代专业版 docx。**若运行环境缺少文档工具或文件写入权限，必须明确报告“文件生成阻塞”和所需补齐条件，不得把 Markdown 当成已完成的专业版文件交付。只有用户明确要求不要文件时，才可改为聊天/Markdown 专业稿。
- **结构化分析内核必须先完整落地**，再渲染客户版文案。没有内核，不得直接写客户版。
- **不因数据档次降档交付物**：数据不足、超窗、冲突或只能取得 C 级数据时，不得把专业版文件降成短评或空框架；必须继续输出专业版文件，并在正文首页、对应分析块和文末附录显式标注“数据档次 / 受限结论 / 需补充材料”。证据不足的结论写为“待核实判断”或“条件性判断”，不得伪装成强结论。
- **研报框架必须执行**：各模块 `SKILL.md` 规定的分析块、研究方法和证伪链必须逐项覆盖。确有不适用项时，必须写明“不适用原因 + 替代观察指标”，不得跳过。
- **专业版完成标准不按字数判定**：保持完整专业版，但禁止为达到篇幅而重复、扩写弱证据或补造内容。终稿必须报告：分析块覆盖率、关键结论证据覆盖率、一手/权威二手来源占比、公式与假设透明度、结论—证据—证伪闭环率、数据缺口披露状态。任一规定分析块缺失，必须列入未完成项。
- 终稿交付前必须自检：文件是否真实生成、模块规定的全部分析块是否齐全、每个关键数字是否带来源和状态、免责声明是否存在。任何一项未完成，必须列入“未完成项”，不能隐去。

## 二、结构化分析内核（通用 schema，各模块继承并具体化）

```yaml
meta:
  标的: ""          # 公司名/题材名/新股名
  代码: ""          # 证券代码（题材类可空）
  市场: ""          # A股 / 港股
  币种: ""          # RMB / HKD
  数据截至: ""      # YYYY-MM-DD
  确定性分级: ""    # 整体 A/B/C，见 data-sourcing.md
  模块: ""          # caijing-xxx

结论:
  事实摘要: ""      # 不含判断的核心事实
  研究判断: ""      # 有证据支撑的方向性结论，专业版与客户版共享
  判断依据: []      # 支撑研究判断的核心数据与事实
  适用边界: []      # 前提、数据限制与方法局限
  关键变量: []      # 后续值得跟踪的 1-3 个公开指标
  反面证据: []      # 与主判断相反或可能推翻判断的证据
  行动建议: null    # 默认空；仅行动安全 Gate 通过后可生成，永不进入客户版

关键数字:            # 卡片和 PPT 的数据来源，5-8 个
  - {指标: "", 数值: "", 对比: "同比/环比/分位/同业", 来源: "", 状态: "已核实|估算"}

分析块:              # docx 正文和 PPT 页面的来源，各模块定义自己的块
  - {标题: "", 要点: [], 支撑数据: [], 确定性: ""}

风险:
  - {风险点: "", 证据: "", 严重度: "高|中|低"}

研究方法:            # 可选，按模块需要填；客户版保留证据化判断，删除交易动作
  逻辑支柱: []        # 3-5 条，每条含 当前证据 / 最新变化 / 证伪条件
  三情景: []          # 乐观 / 基准 / 悲观，只写变量和触发条件，不写客户版动作
  同业四分位: []      # 指标、同业池、P75/Median/P25、当前所处位置
  验证链: []          # 后续要跟踪的财报、订单、价格、政策、风险节点
  证伪条件: []        # 任何核心判断被推翻的条件
  管理与资本配置: []  # 激励、目标兑现、再投资/并购/分红回购/稀释、单位经济
  预期与调整项桥: []  # 预期分布/修正、reported→adjusted、指引兑现记录
  正常化与交叉验证: [] # 正常化盈利、摊薄股本、终值占比、SOTP/多方法冲突
  模型校准: []        # 适用范围、本土局限、误报解释、一手验证
  产业暴露: []        # 纯度、收入弹性、利润弹性、竞争优势、估值反映
  统计有效性: []      # 样本、基准率、置信区间、预测时点、滚动复盘
  结论证明:            # caijing-fundamental 强制；其他模块可选
    - 研究问题: ""
      关联分析块: ""
      原始数据: []
      口径调整: ""
      公式或拆解: ""
      计算结果: ""
      历史坐标: ""
      同业坐标: ""
      研究判断: ""
      反面证据: []
      证伪条件: []
      状态: "完整|断裂"
      断点: ""
      缺失数据: []
  框架外发现:          # caijing-fundamental 强制扫描；其他模块可选
    - 事项: ""
      触发证据: ""
      标准框架未覆盖原因: ""
      重要性命中项: []
      处理: "新增正文分析块|附录观察"
      关联证明链: ""
      对原结论影响: ""

```

`caijing-fundamental` 必须填充 `研究方法.结论证明` 并记录 `研究方法.框架外发现` 扫描结果；其他模块可选，不因本次升级改变既有默认工作流。

## 三、三形态渲染规范

### ① 文档（docx，专业版主场）

- 研报级结构，具体大纲各模块 SKILL.md 定义；篇幅服从信息密度与证据充分度，不设机械字数下限。客户版、短版或用户明确要求摘要时另按需求控制篇幅。
- 首页必有「一页概要 / 投资要点」：事实摘要 + 研究判断 + 判断依据 + 关键数字表 + 关键变量——让使用者 30 秒抓住全貌。
- 每个关键数字带来源脚注；每张图/表编号并附来源行；文末数据附录（来源清单 + 确定性分级 + 数据缺口声明）+ 固定免责声明。
- **视觉规范唯一事实源见 [docx-visual-spec.md](docx-visual-spec.md)**（本土券商研报骨架 + 国际卖方图表纪律 + 炭黑金配色 + 思源宋体/MiSans 字体）；渲染 docx 时必读，不得自定另一套视觉。
- 用文档处理能力生成 `.docx`。若环境中存在 `documents:documents` skill，必须先读取并使用；否则使用本地可用的 docx 库生成。不得只把正文贴在聊天窗口里。

### ② 卡片图 + 文案（客户版主场）

- **卡片**：字段从 `关键数字` + `结论.研究判断` + `结论.判断依据` + `风险` 取；功能位与固定件见 card-components.md。
- **文案**：150–300 字，微信直接转发口吻，结构固定：
  1. 一句话研究判断（来自内核，非新写）
  2. 2–3 个关键数字（带对比口径）
  3. 1 条风险提示
  4. 固定免责声明
- 文案必须过 compliance.md 客户版 checklist，做到**复制即用**。

### ③ 汇报 PPT（按需件：仅用户明确要"汇报/PPT/deck/幻灯"时才出）

- **触发**：PPT **不随多模块组合自动产出**；只在用户明确要求时生成。多模块组合默认仍是逐模块专业版 docx（或合并摘要）。
- **视觉规范唯一事实源见 [ppt-visual-spec.md](ppt-visual-spec.md)**；生成 PPT 前必读，不得从 `docs/` 读取执行规则或另起一套视觉。
- **默认形态 = 真 .pptx（可在 PowerPoint/WPS 编辑）**：使用 `assets/deck-template.mjs` 的 pptxgenjs renderer，默认受众为投顾内部研究汇报。renderer 接受结构化 deck JSON，自适应主题、七类固定版式和可选 Pipeline；生成后用 PPT 工具链重压并逐页渲染 QA。
- **零依赖备选 = HTML 翻页 deck** `assets/template-deck.html`（← → / 圆点 / 滚轮 / 触屏翻页，`Ctrl/Cmd+P` 存 PDF）：用户没装 PowerPoint、或只要浏览器演示/PDF 时用。
- 真 `.pptx` 与 HTML deck 必须消费同一份结构化内核；不得各写一份分析。固定版式为研究命题封面、综合结论、投资逻辑与核心矛盾、基本面/财报、估值坐标、风险证伪、免责声明；产业链可插入 Pipeline。
- 图片为可选叙事资产：Codex 可用 imagegen 增强，Claude 使用用户供图或完整无图版式。图片不可得、质量不足或挤压重要信息时必须无图降级，不保留空槽。
- 默认专业研究口径；客户合规口径保留研究判断和情景条件，删除交易动作与个性化资金内容。
- 颜色不固定；同一 deck 仅一个 `accent`，`up/down` 只用于真实方向数据。详细字体、网格、留白、密度、图片比例、主题令牌与 QA 见 PPT 视觉事实源。

## 四、受众口径与形态切换（渲染时执行，不改内核）

客户合规口径渲染时：

1. 保留 `结论.研究判断`、`判断依据`、`适用边界`、`关键变量` 和 `反面证据`。
2. 永久丢弃 `结论.行动建议`。
3. `研究方法.逻辑支柱 / 三情景 / 验证链 / 证伪条件` 可以保留方向性研究含义，但改写为可验证的条件、反面证据和后续观察指标，不转化为交易动作。
4. 删除目标价、收益承诺、个性化仓位、金额、杠杆、申购档位和操作时点。
5. 逐句过 compliance.md 与 compliance-rendering.md checklist。

受众口径与交付形态彼此独立：客户版可以是 docx、卡片+文案或 PPT；专业研究也可以按用户要求生成卡片或 PPT。所有形态必须使用同一内核、同一数据截至日和同一版本。

## 五、视觉模板进度与交付规则

- **docx 视觉已定稿（2026-07-10）**：规范见 [docx-visual-spec.md](docx-visual-spec.md)，渲染专业版 docx 时按其执行。
- **卡片视觉已定稿并落地（2026-07-10）**：见 [card-components.md](card-components.md)「墨账本 Swiss」——墨黑招牌带 + 冷白账本底 + 红绿只给涨跌数据；模板 `assets/template-card.html`（七模块完整卡片）已建。**默认交付填好数据的 HTML + 提醒截图，不出 PNG**。
- **PPT 视觉已定稿并升级为通用自适应系统（2026-07-10）**：**按需件**，默认真 .pptx（pptxgenjs，参考 `assets/deck-template.mjs`），零依赖备选 HTML 翻页 deck `assets/template-deck.html`；唯一执行事实源见 [ppt-visual-spec.md](ppt-visual-spec.md)。

**各形态交付规则（三形态视觉均已定稿）：**

- 单模块默认交付：专业研究 `.docx`（按 docx-visual-spec.md 渲染）+ 结构化内核摘要。客户合规版按用户指定生成 docx、卡片+文案或 PPT；未指定形态时默认卡片+文案。文档生成工具不可用或文件写入失败时，必须列为“文件生成阻塞”；只有用户明确不要文件时才改为聊天或 Markdown 专业稿。
- 客户版卡片（按需）交付「填好数据的 HTML 卡片（复制 template-card.html 对应 .card 填内核）+ 文案 + 提醒截图」，不出 PNG。
- PPT 为**按需件**（仅用户点名"汇报/PPT/deck"才出，多模块不自动出）：默认真 .pptx（pptxgenjs 自适应内部研究版，见 §三③与 ppt-visual-spec.md）；用户没装 PowerPoint 或只要 PDF 时给 HTML 翻页 deck。
- 若环境无法写文件，先说明阻塞，再按现有素材产最小可用版。
