# 双版本输出层（内核 schema + 三形态渲染规范）

## 一、地基原则：一份内核、三形态渲染

每次分析只产出**一份「结构化分析内核」**；docx、卡片、PPT 都是这份内核的不同渲染视图。**绝不允许**为不同形态分别撰写分析——那会导致三处不一致、改一处漏两处。修改分析时改内核，再重新渲染。

## 二、结构化分析内核（通用 schema，各模块继承并具体化）

```yaml
meta:
  标的: ""          # 公司名/题材名/新股名
  代码: ""          # 证券代码（题材类可空）
  市场: ""          # A股 / 港股
  币种: ""          # RMB / HKD
  数据截至: ""      # YYYY-MM-DD
  确定性分级: ""    # 整体 A/B/C，见 data-sourcing.md
  模块: ""          # caijing-xxx

结论:
  客观概括: ""      # 一句话纯事实概括，客户版可直接用
  专业判断: ""      # 一句话倾向判断，仅专业版；客户版渲染时丢弃
  多空关键变量: []  # 后续值得跟踪的 1-3 个变量

关键数字:            # 卡片和 PPT 的数据来源，5-8 个
  - {指标: "", 数值: "", 对比: "同比/环比/分位/同业", 来源: "", 状态: "已核实|估算"}

分析块:              # docx 正文和 PPT 页面的来源，各模块定义自己的块
  - {标题: "", 要点: [], 支撑数据: [], 确定性: ""}

风险:
  - {风险点: "", 证据: "", 严重度: "高|中|低"}

数据缺口: []         # 没拿到的关键数据，输出时如实声明
```

## 三、三形态渲染规范

### ① 文档（docx，专业版主场）

- 研报级结构，具体大纲各模块 SKILL.md 定义；篇幅指引 2000–4000 字（基本面取上限，事件类取下限）。
- 首页必有「一页概要」：客观概括 + 专业判断 + 关键数字表 + 多空关键变量——让投顾 30 秒抓住全貌。
- 每个关键数字带来源脚注；文末数据附录（来源清单 + 确定性分级 + 数据缺口声明）+ 固定免责声明。
- 用 docx skill 生成。

### ② 卡片图 + 文案（客户版主场）

- **卡片**：字段从 `关键数字` + `结论.客观概括` + `风险` 取；功能位与固定件见 card-components.md。
- **文案**：150–300 字，微信直接转发口吻，结构固定：
  1. 一句话客观概括（来自内核，非新写）
  2. 2–3 个关键数字（带对比口径）
  3. 1 条风险提示
  4. 固定免责声明
- 文案必须过 compliance.md 客户版 checklist，做到**复制即用**。

### ③ 汇报 PPT（多模块组合主场）

- 仅在多模块组合时产出；单模块不出 PPT。
- 结构：封面 → 总览页（各模块结论汇总表）→ 每模块 2–4 页（从各自内核的分析块渲染）→ 风险汇总页 → 免责页。
- 默认专业口径；用户指定客户口径时，全部页面按口径切换规则降级。
- 用 pptx skill 生成。

## 四、口径切换规则（渲染时执行，不改内核）

客户口径渲染时：

1. 丢弃 `结论.专业判断`，只用 `客观概括`。
2. `多空关键变量` 改写为"后续可关注的指标"。
3. 所有倾向性形容词降级为数据陈述（对照 compliance.md 降级转换表）。
4. 逐句过客户版 checklist。
