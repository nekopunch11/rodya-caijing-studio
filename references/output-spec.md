# 双版本输出层（内核 schema + 三形态渲染规范）

## 一、地基原则：一份内核、三形态渲染

每次分析只产出**一份「结构化分析内核」**；docx、卡片、PPT 都是这份内核的不同渲染视图。**绝不允许**为不同形态分别撰写分析——那会导致三处不一致、改一处漏两处。修改分析时改内核，再重新渲染。

## 零、交付硬门槛（先执行）

- 用户触发任一 `caijing-*` 模块且未明确要求“只在聊天里回答”时，**专业版默认必须生成文件成品**：单模块生成 `.docx`，多模块组合才生成 PPT。聊天区只能交付摘要、文件路径、数据截至日、确定性分级、数据缺口和未完成项。
- **客户版默认不出、按需生成**：单模块默认只交付专业版 `.docx`；客户版（卡片图+文案）仅当用户要转客户或明确要客户版时才渲染，届时逐句过 compliance-rendering.md 降级。**交付专业版后必须主动提醒使用者：如需转客户可另出客户版。**（`caijing-ipo-hk` 对客模式同为按需触发。）
- **不得用几段 Markdown 短评替代专业版 docx。**若运行环境缺少文档工具或文件写入权限，必须明确报告“文件生成阻塞”和所需补齐条件，不得把 Markdown 当成已完成的专业版文件交付。只有用户明确要求不要文件时，才可改为聊天/Markdown 专业稿。
- **结构化分析内核必须先完整落地**，再渲染客户版文案。没有内核，不得直接写客户版。
- **不因数据档次降档交付物**：数据不足、超窗、冲突或只能取得 C 级数据时，不得把专业版文件降成短评或空框架；必须继续输出专业版文件，并在正文首页、对应分析块和文末附录显式标注“数据档次 / 受限结论 / 需补充材料”。证据不足的结论写为“待核实判断”或“条件性判断”，不得伪装成强结论。
- **研报框架必须执行**：各模块 `SKILL.md` 规定的分析块、研究方法和证伪链必须逐项覆盖。确有不适用项时，必须写明“不适用原因 + 替代观察指标”，不得跳过。
- **专业版篇幅底线**：自用专业版研报或专业结果默认不少于 5000 字；为避免凑字数、幻觉或证据不足可适当降低，但仍不得低于 4000 字。只有用户明确要求“短版/聊天回答/只要摘要”时，才可低于该底线。
- 终稿交付前必须自检：文件是否真实生成、九/七个分析块是否齐全、每个关键数字是否带来源和状态、免责声明是否存在。任何一项未完成，必须列入“未完成项”，不能隐去。

## 二、结构化分析内核（通用 schema，各模块继承并具体化）

```yaml
meta:
  标的: ""          # 公司名/题材名/新股名
  代码: ""          # 证券代码（题材类可空）
  市场: ""          # A股 / 港股
  币种: ""          # RMB / HKD
  数据截至: ""      # YYYY-MM-DD
  确定性分级: ""    # 整体 A/B/C，见 data-sourcing.md
  模块: ""          # caijing-xxx

结论:
  客观概括: ""      # 一句话纯事实概括，客户版可直接用
  专业判断: ""      # 一句话倾向判断，仅专业版；客户版渲染时丢弃
  多空关键变量: []  # 后续值得跟踪的 1-3 个变量

关键数字:            # 卡片和 PPT 的数据来源，5-8 个
  - {指标: "", 数值: "", 对比: "同比/环比/分位/同业", 来源: "", 状态: "已核实|估算"}

分析块:              # docx 正文和 PPT 页面的来源，各模块定义自己的块
  - {标题: "", 要点: [], 支撑数据: [], 确定性: ""}

风险:
  - {风险点: "", 证据: "", 严重度: "高|中|低"}

研究方法:            # 可选，按模块需要填；客户版渲染时只保留客观变量
  逻辑支柱: []        # 3-5 条，每条含 当前证据 / 最新变化 / 证伪条件
  三情景: []          # 乐观 / 基准 / 悲观，只写变量和触发条件，不写客户版动作
  同业四分位: []      # 指标、同业池、P75/Median/P25、当前所处位置
  验证链: []          # 后续要跟踪的财报、订单、价格、政策、风险节点
  证伪条件: []        # 任何核心判断被推翻的条件

数据缺口: []         # 没拿到的关键数据，输出时如实声明
```

## 三、三形态渲染规范

### ① 文档（docx，专业版主场）

- 研报级结构，具体大纲各模块 SKILL.md 定义；自用专业版默认不少于 5000 字，证据不足或防止凑字数时可压缩但不得低于 4000 字。客户版、短版或用户明确要求摘要时另按需求控制篇幅。
- 首页必有「一页概要 / 投资要点」：客观概括 + 专业判断 + 关键数字表 + 多空关键变量——让投顾 30 秒抓住全貌。
- 每个关键数字带来源脚注；每张图/表编号并附来源行；文末数据附录（来源清单 + 确定性分级 + 数据缺口声明）+ 固定免责声明。
- **视觉规范唯一事实源见 [docx-visual-spec.md](docx-visual-spec.md)**（本土券商研报骨架 + 国际卖方图表纪律 + 炭黑金配色 + 思源宋体/MiSans 字体）；渲染 docx 时必读，不得自定另一套视觉。
- 用文档处理能力生成 `.docx`。若环境中存在 `documents:documents` skill，必须先读取并使用；否则使用本地可用的 docx 库生成。不得只把正文贴在聊天窗口里。

### ② 卡片图 + 文案（客户版主场）

- **卡片**：字段从 `关键数字` + `结论.客观概括` + `风险` 取；功能位与固定件见 card-components.md。
- **文案**：150–300 字，微信直接转发口吻，结构固定：
  1. 一句话客观概括（来自内核，非新写）
  2. 2–3 个关键数字（带对比口径）
  3. 1 条风险提示
  4. 固定免责声明
- 文案必须过 compliance.md 客户版 checklist，做到**复制即用**。

### ③ 汇报 PPT（多模块组合主场，HTML 翻页 deck）

- 仅在多模块组合时产出；单模块不出 PPT。
- **形态 = 单文件 HTML 翻页 deck（墨账本 Swiss 皮），零依赖**：模板 `assets/template-deck.html`（16:9；← → / 圆点 / 滚轮 / 触屏翻页，`Ctrl/Cmd+P` 打印成 PDF）。复制 `.slide` 填内核数据即用；视觉与卡片同源（招牌带 + 墨黑 + 红绿只给涨跌数据、不进招牌带），**不另造视觉、不依赖 pptx skill / PowerPoint**。
- 结构（版式库）：封面 → 总览页（各模块结论汇总表）→ 每模块页（数据大字报 / 左右分栏，套卡片签名图形）→ Pipeline（产业链）/ 对比页按需 → 风险汇总页 → 免责页。
- 默认专业口径；用户指定客户口径时全部页面按口径切换规则降级（去「专业判断 / 三情景」等）。
- 交付：给 HTML 文件 + 提醒「浏览器打开演示，或 `Ctrl/Cmd+P` 存成 PDF」。

## 四、口径切换规则（渲染时执行，不改内核）

客户口径渲染时：

1. 丢弃 `结论.专业判断`，只用 `客观概括`。
2. `多空关键变量` 改写为"后续可关注的指标"。
3. `研究方法.逻辑支柱 / 三情景 / 验证链 / 证伪条件` 只提取为"后续可关注的指标/公开信息提示"，不展示"增强/削弱/乐观/悲观/强瓶颈"等倾向判断。
4. 所有倾向性形容词降级为数据陈述（对照 compliance.md 与 compliance-rendering.md）。
5. 逐句过客户版 checklist。
