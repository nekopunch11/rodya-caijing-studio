# 基本面结论证明协议

> 本文件是 `caijing-fundamental` 判断性结论的唯一证明规则。安全、合规、取数与时效要求仍服从共享层中优先级更高的规则。

## 一、适用范围

十个分析块是最低覆盖，不是封闭框架。所有基本面专业版默认执行本协议，不设置“标准版 / 深研版”双档。

## 二、判断性结论识别

含“强、弱、改善、恶化、领先、落后、可持续、存在护城河、资本回报有效、成长质量高”等定性的命题，或会进入首页研究判断、护城河评分、逻辑支柱和风险定性的表述，必须建立证明链。

纯事实可以不计算，但必须带来源、口径和截至日。无法由证据链支持的形容词不是研究结论。

## 三、结论证明链

统一链条为：

`研究问题 → 原始数据 → 口径调整 → 公式或拆解 → 计算结果 → 历史坐标 → 同业坐标 → 研究判断 → 反面证据 → 证伪条件`

每条链必须记录：编号、关联分析块、原始数据的来源等级与页码、状态（完整 / 断裂）、断点和缺失数据。历史坐标或同业坐标确实不适用时，必须记录不适用原因和替代坐标。

### 最小结构

```yaml
- 编号: "P-01"
  研究问题: ""
  关联分析块: ""
  原始数据: []
  口径调整: ""
  公式或拆解: ""
  计算结果: ""
  历史坐标: ""
  同业坐标: ""
  研究判断: ""
  反面证据: []
  证伪条件: []
  状态: "完整|断裂"
  断点: ""
  缺失数据: []
```

## 四、结论 Gate

只有状态为“完整”的链可以输出研究判断。推导链状态为“断裂”时，停止对受影响事项下定性结论；只保留已核实事实、断点、缺失数据、受影响结论和替代观察指标。

不得用条件性“偏强 / 偏弱”维持结论完整感，不得用外部预测值补成公司实际数据，也不得用总公司指标冒充未披露的分部或单位经济。

## 五、框架外扫描

完成标准分析块后必须扫描：

1. 与典型同业不同的资产、负债或收入结构；
2. 能解释大部分利润但未被标准框架捕捉的变量；
3. 使常规指标失真的会计口径、交易结构或监管安排；
4. 足以改变原逻辑的新业务、重大诉讼、关联交易、技术路线或资本动作；
5. 财务数据与商业现实不匹配的异常。

没有重大事项也要记录“已扫描，未发现通过重要性 Gate 的事项”。

## 六、重要性 Gate

框架外事项满足任一条件时，新增正文分析块并建立独立证明链：

- 可能改变收入、利润或现金流的主要来源；
- 可能改变资本结构、控制权或每股价值；
- 可能推翻护城河评分或投资逻辑支柱；
- 常规财务指标因它而失真；
- 公司、审计机构、监管机构或管理层讨论将其列为重大事项。

未通过重要性 Gate 的事项进入附录观察清单，不扩写正文。

## 七、行业适配器覆盖

允许覆盖默认适配器，但必须写明：默认指标的失真原因、采用的替代指标、替代指标如何连接研究判断，以及替代方法的局限。框架外事项不降低证据标准。

## 八、正文与附录

正文展示决定结论的关键推导，包括研究问题、核心原始数据、必要口径调整、公式或拆解、历史与同业坐标、研究判断、反面证据和证伪条件。

附录保存原始数据、口径调整、完整同业池、重复计算、来源页码、断裂链和补数清单。正文、附录和首页使用同一证明链编号。

## 九、首页反向审查

首页每一句判断必须能定位到一条或多条状态为“完整”的证明链。无法反查的判断删除或降级为事实；只引用断裂链的判断不得进入首页。

## 十、完成自检

- 判断性结论证明覆盖率为 100%；
- 强判断关联的断裂链为 0；
- 关键计算可由报告中的原始数据、口径和公式复算；
- 历史与同业坐标齐全，或有不适用原因和替代坐标；
- 每条核心逻辑同时有支持证据、反面证据和证伪条件；
- 框架外扫描已记录；
- 行业适配器采用或覆盖理由已披露。
