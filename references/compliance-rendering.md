# 合规渲染规则（专业版 -> 客户版）

定位：客户版是可转发材料，不是研究员内部判断。渲染客户版时，先按 `compliance.md` 做红线检查，再按本文件降级表达。

## 一、客户版禁用表达升级

除 `compliance.md` 禁用词外，客户版还避免以下表达：

- 整体评级、总评、强/弱、好/差、便宜/贵、低估/高估
- 值得打、值得关注、首选、最受益、受益最大
- 红灯/绿灯作为行动暗示
- 大概率、确定性、修复、反转、兑现、爆发

## 二、降级转换

| 专业版表达 | 客户版表达 |
|---|---|
| 基本面偏强/偏弱 | 列 ROE、现金流、增长、负债等事实 |
| 估值低/高 | 当前估值处于近 N 年 X% 分位 |
| 风险红/黄/绿 | 公开信息提示项 N 项，其中重点关注 X 项 |
| 破发风险高/中/低 | 统计风险提示档位：偏高/中性/偏低 |
| 值不值得打 | 发行结构与历史样本统计如下 |
| 受益标的 Top3 | 相关业务收入占比较高的代表公司示例 |
| 利润池最好环节 | 从价值量和集中度看，利润较多沉淀在 X 环节 |

## 三、客户版固定结构

1. 一句话事实概括。
2. 2-3 个关键数字，带来源或截至日。
3. 1 条风险或数据缺口。
4. 固定免责声明。

客户版不得出现“专业判断”字段；不得引入内核之外的新观点。

## 四、打新客户版特别规则

- 可写“统计风险提示档位”，不可写“建议申购/放弃/谨慎参与”。
- 必须写样本窗口、样本数或“样本不足”。
- 孖展超购、暗盘、回拨、保荐人胜率不得单独推出结论。
- “入场费”可作为发行要素，不得引出资金分配建议。

## 五、产业链客户版特别规则

- 公司名单用“代表公司示例”，不写“股票池”。
- 若列公司，必须带相关业务收入占比或“未披露，待核实”。
- 不按“最值得关注”排序；默认按产业环节排列。

## 六、发布前检查

- 是否移除了所有倾向判断？
- 是否把评级/档位解释为统计或公开信息提示？
- 是否有来源、日期、风险和免责声明？
- 客户读完是否只得到信息，而不是动作？
