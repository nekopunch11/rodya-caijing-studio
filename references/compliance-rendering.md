# 客户合规渲染规则

定位：客户合规版不是专业版阉割稿。它消费同一结构化研究内核，保留有证据支撑的方向性研究判断，只移除交易动作、个性化资金建议、收益承诺和未经批准的机构评级。

## 一、必须保留

1. 一句话研究结论；
2. 支撑结论的判断依据，包括 2–3 个关键数字或公开事实；
3. 适用边界或数据缺口；
4. 至少 1 条风险或反面证据；
5. 1–3 个后续观察指标；
6. 数据截至日、来源和固定免责声明。

不得删除 `结论.研究判断`，不得把客户版降成只有数字和“供参考”的信息清单。

## 二、必须移除

- `结论.行动建议` 的全部内容；
- 买卖、申购、放弃、持仓和退出指令；
- 具体仓位、金额、股数、融资、杠杆和下单时间；
- 目标价、收益承诺、精确涨跌预测和未经校准的事件概率；
- “首选、重仓候选、值得打、打满、上车”等动作暗示；
- 未经所属机构批准的正式评级、建议或承诺表述。

## 三、专业判断的客户化表达

| 内核研究判断 | 客户合规表达 |
|---|---|
| 基本面偏强，但现金流转弱 | 盈利与竞争位置仍有支撑，但本期现金转换走弱，需要继续验证回款和应收变化 |
| 估值偏高 | 当前估值位于近 N 年较高分位，隐含增长预期高于历史中枢 |
| 财报超预期但质量一般 | 相对一致预期/公司指引，利润高出 X%，但现金流和扣非表现未同步改善 |
| 风险较高 | 已识别 N 项重点风险，其中 X 可能影响盈利质量或资产安全 |
| 利润池集中在 X 环节 | 从单位价值量、毛利率、集中度和议价权看，利润主要沉淀在 X 环节 |
| 破发统计风险偏高 | 在相同板块、定价和发行结构样本中，首日表现相对不利，样本窗口为…… |

## 四、模块特别规则

- **估值**：可以写高/低分位、隐含预期偏积极/保守及其证据；不能据此给买卖动作。
- **排雷**：可以判断风险集中度和严重性；不能写爆雷预测、做空依据或处置动作。
- **产业链**：可以判断利润沉淀、供给约束和业务暴露；公司名单按产业环节或业务相关度排列，不写推荐排序。
- **A股/港股打新**：可以写统计风险档位、样本和发行结构判断；客户版不得展示申购档位、金额、融资或暗盘处置动作。
- **港股打新**：行动层的三档方案、推荐和执行清单永不进入客户合规版。

## 五、形态适配

- 客户版 docx、卡片、文案和 PPT 均使用本规则，不把“客户版”等同于“卡片”。
- 卡片空间不足时优先保留研究判断、关键证据和最高优先级风险；完整边界与来源放配套文案。
- 客户 PPT 可以保留研究判断和情景条件，但不能出现操作动作页。

## 六、发布前检查

- 是否保留了清晰、可验证的研究结论？
- 结论是否带证据、边界、风险和观察指标？
- 是否删除了行动建议和个性化资金内容？
- 是否有来源、日期和免责声明？
- 客户读完是否得到专业判断，但仍需自行作出投资决定？
