# 数据档次与补充提示规则（联网失败 / 数据缺失 / 口径冲突）

定位：防止在缺少关键数据时硬编结论，同时避免把专业交付物降成空框架。任何模块取数遇到缺口时，按本文件标注证据档次、结论边界和补充材料，再按 `data-sourcing.md` 标来源与确定性。

## 一、总规则

- **没有一手材料，不出强结论**：定期报告、公告、招股书、发行公告缺失时，仍输出专业版文件，但相关判断必须标为“待核实判断/条件性判断”，并列出需要补充的原文材料；不得把弱证据写成确定评级、分档或超预期结论。
- **行情类过期，保留文件但限制结论强度**：估值、分位、破发统计、孖展数据若无法确认数据截至日，正文必须标“时效存疑”，客户版只保留客观日期和缺口提示，不输出档位/评级。
- **C 级数据不能独立支撑强结论**：若核心结论主要依赖 C 级估算，结论字段写“待核实线索/条件性判断”，不得写“偏强/偏弱/高风险/低风险”等强判断。
- **冲突数据并列，不擅自取舍**：不同来源数字冲突时列出来源、口径、日期；若无法判断，使用“存在口径差异，暂不据此下结论”。
- **用户截图只能当线索**：截图未能追溯到公告/报告原文时，标 C 级；关键数字不得只由截图支撑。

## 二、模块最低证据要求

| 模块 | 最低材料 | 缺失时必须提示 | 缺失时禁止写成强结论 |
|---|---|---|---|
| 基本面 | 近 3 年年报 + 最新一期财报 | 缺少年度、期间、来源与受影响的分析块 | 护城河总分、ROIC-WACC 差值、确定性专业判断 |
| 财报 | 本期财报原文 | 未核验财报期、缺少科目、需补原文链接 | 超预期判定、现金流含金量强结论 |
| 估值 | 当前估值指标 + 历史序列或同业表 | 数据日期、同业池缺口、分位口径限制 | 历史分位、反向 DCF、贵/不贵强判断 |
| 排雷 | 年报附注 + 审计意见 + 监管/公告记录 | 未覆盖的附注、公告与监管检索范围 | 总评灯、一票否决结论、模型评分强判断 |
| 产业链 | 产业口径来源 + 分部收入或公司披露 | 产业口径、分部收入、纯度证据缺口 | 纯度 Top 排序、利润池强结论 |
| A股打新 | 招股意向书/发行公告 | 招股材料、发行要素、样本窗口缺口 | 破发统计档位、申购日客户版强判断 |
| 港股打新 | 招股书 + 招股区间 + 发售结构 | 招股书、孖展时点、同类样本缺口 | 同类组破发率、孖展热度判断、风险档位强结论 |

## 三、数据档次提示模板

### 数据不足说明

> 当前缺少 {关键材料}，{模块核心判断} 只能形成条件性判断，不能写成强结论。本文仍按专业框架完成分析，并将相关段落标为 {A/B/C 或 待核实}；后续需补充 {所需材料} 后复核结论。

### 冲突数据说明

> 关于 {指标}，当前来源存在差异：{来源 A} 为 {数值 A}，{来源 B} 为 {数值 B}。因口径/日期未能统一，暂不将该指标作为结论依据。

### 截至日不足说明

> {指标} 未能确认数据截至日，本文仅列为参考信息，不纳入客户版关键数字。

## 四、执行顺序

1. 先判定是否满足模块最低材料。
2. 不满足时，仍执行模块 `SKILL.md` 的完整分析框架，但把受影响的公式/评分/分档改写为“无法形成强结论 + 需要补充的材料 + 可观察替代指标”。
3. 满足时，再执行模块 `SKILL.md` 与 `formulas-and-thresholds.md` 的完整公式、评分和分档。
4. 交付末尾列出 `数据缺口` 和 `补充要求`，不要把缺口藏在正文里。
