# 取数降级规则（联网失败 / 数据缺失 / 口径冲突）

定位：防止在缺少关键数据时硬编结论。任何模块取数遇到缺口时，先按本文件降级，再按 `data-sourcing.md` 标来源与确定性。

## 一、总规则

- **没有一手材料，不出强结论**：定期报告、公告、招股书、发行公告缺失时，只输出“待核实体检框架 + 已知公开信息”，不输出专业判断、评级、分档。
- **行情类过期，估值/打新降级**：估值、分位、破发统计、孖展数据若无法确认数据截至日，客户版不渲染卡片，只给文字说明。
- **C 级数据不能独立支撑结论**：若核心结论主要依赖 C 级估算，结论字段写“待核实线索”，不得写“偏强/偏弱/高风险/低风险”。
- **冲突数据并列，不擅自取舍**：不同来源数字冲突时列出来源、口径、日期；若无法判断，使用“存在口径差异，暂不据此下结论”。
- **用户截图只能当线索**：截图未能追溯到公告/报告原文时，标 C 级；关键数字不得只由截图支撑。

## 二、模块最低证据要求

| 模块 | 最低材料 | 缺失时允许输出 | 缺失时禁止输出 |
|---|---|---|---|
| 基本面 | 近 3 年年报 + 最新一期财报 | 公司速览、待补数据清单 | 护城河总分、ROIC-WACC 差值、专业判断 |
| 财报 | 本期财报原文 | 财报阅读提纲、需核实指标 | 超预期判定、现金流含金量结论 |
| 估值 | 当前估值指标 + 历史序列或同业表 | 估值框架、适用指标说明 | 历史分位、反向 DCF、贵/不贵判断 |
| 排雷 | 年报附注 + 审计意见 + 监管/公告记录 | 红旗核查清单 | 总评灯、一票否决结论、模型评分 |
| 产业链 | 产业口径来源 + 分部收入或公司披露 | 链条框架、待核实标的表 | 纯度 Top 排序、利润池结论 |
| A股打新 | 招股意向书/发行公告 | 发行要素待核实清单 | 破发统计档位、申购日客户版 |
| 港股打新 | 招股书 + 招股区间 + 发售结构 | 招股要素待核实清单 | 同类组破发率、孖展热度判断、风险档位 |

## 三、输出降级模板

### 数据不足说明

> 当前缺少 {关键材料}，无法完成 {模块核心判断}。以下仅为待核实体检框架，后续取得 {所需材料} 后再补充结论。

### 冲突数据说明

> 关于 {指标}，当前来源存在差异：{来源 A} 为 {数值 A}，{来源 B} 为 {数值 B}。因口径/日期未能统一，暂不将该指标作为结论依据。

### 截至日不足说明

> {指标} 未能确认数据截至日，本文仅列为参考信息，不纳入客户版关键数字。

## 四、执行顺序

1. 先判定是否满足模块最低材料。
2. 不满足时，按本文件降级，不进入公式/评分/分档。
3. 满足时，再执行模块 `SKILL.md` 与 `formulas-and-thresholds.md`。
4. 交付末尾列出 `数据缺口`，不要把缺口藏在正文里。
