# 数据卡片层 · 视觉规范（客户版主场，v0.5 · 2026-07-15）

> 定位：客户版对客卡片（微信对话 / 朋友圈 / 小红书信息流）。视觉身份 = **「墨账本 Swiss」**。改视觉只改本文件与 `assets/`，七模块全生效。
> 卡片是可转发对客物料，内容必须过 `compliance.md` + `compliance-rendering.md` 客户版 checklist；卡片默认按需生成（见 output-spec.md）。

## 一、视觉身份

**墨账本 Swiss** —— 瑞士国际主义纪律 + 招牌带 + 墨黑账本底：

- **Swiss 硬纪律**：严格网格、直角发丝线、极致字号对比、**「越大越细」**、克制而非海报式留白、锁死配色不许自定义 hex。
- **招牌带（标的条）**：每张卡顶部一条满幅品牌色带，等宽字体压 标的 · 代码 · 市场 · 数据截至，像行情终端 / 研报抬头；七模块共用 = 强识别，也是 docx 首页问责块的卡片版。
- **颜色只准出现在两个地方**：① 品牌色（招牌带）② 市场数据（红涨绿跌）。卡片主体是墨黑 × 冷白账本底。
- **数据台密度（非海报留白）**：财经受众要信息量。**控制留白比例、收小夸张巨号数字、把关键数字铺成紧凑网格（发丝线分隔）**；目标是"研报速览 / 行情快照"的密度，密而有序 ≠ 杂乱，靠网格和发丝线撑层级。参照 guizang 的 Swiss 是"AI 工具测评"的海报式留白，不适用于财经，本 skill 一律走数据台密度。

**密度执行铁律：**

- 正文不得用 `justify-content: space-between` 把模块均分到整张卡；区块使用 20–32 px 固定纵向间距，剩余弹性只能留在最后一个内容区之后。
- 头号数字服务于判断，不得独占大面积；主视觉须同时给出结论状态或关键比较值。
- 关键数字格中的每一项优先写为“数值 + 对比”（同比 / 环比 / 较年末 / 历史分位 / 同业位置）；没有可靠对比数据时明确标“对比待补”，不得补造方向、变化率或迷你图表。
- 研究型客户卡必须把“研究判断”与其证据相邻呈现；估值卡强制使用“支撑证据 / 反面证据或待验证项”双栏，其他模块在资料足够时同样适用。

## 二、配色（锁死，不许自定义 hex）

| 用途 | 色值 |
|---|---|
| 纸底 | 冷白 `#FFFFFF` |
| 墨（文字 / 发丝线 / 图形 / 招牌带） | `#111111` |
| 次级文字 | `#888888` |
| 内部网格发丝线 | `#EDEDED` |
| 空态填充（进度 / 格子未达部分） | `#DCDCDC` |
| 涨（红） | `#C0392B` |
| 跌（绿） | `#1E9E5A` |

**铁律：**

- **红 / 绿只用于涨跌数据与重点数值强调，永不进招牌带、永不做装饰色块。** 红涨绿跌锁 A 股 / 港股惯例，任何图表不得反用。
- **品牌色 = 墨黑 `#111`（定稿）**，只出现在招牌带（及必要的 footer 分隔线），不进数据区。
- 全卡除红 / 绿数据外无第二彩色。

## 三、字体

- 中文无衬线：思源黑体 / MiSans（回退微软雅黑）——Swiss 骨架。
- 数字：tabular figures 等宽。
- 元数据 / 招牌带 / footer：等宽（mono），终端感。
- **「越大越细」**：大号关键数字用细字重（200–300），小标签用中 / 粗（500–600）+ 大字距。

## 四、画板

- **主：竖版 3:4，1080×1440**（微信对话 / 朋友圈 / 小红书通吃）。
- 备（公众号封面对，后续需要再做）：1:1 `1080×1080`、21:9 `2100×900`。

## 五、通用版式（自上而下，锁定）

1. **招牌带**：满幅墨黑 `#111` 底、白字。左 标的名 + 等宽代码，右 模块名；下一行等宽小字 市场 · 数据截至 YYYY-MM-DD。
2. **主视觉区**：模块签名图形 + 头号关键数字（越大越细，重点数标红）。
3. **关键数字区**：**5–8 条**（对齐 docx 关键数字），2–3 列紧凑网格 + 直角发丝线分隔，小标签 + 中号数值 + 一行可追溯比较口径，tabular；只有真实涨跌数据才染红/绿。头号指标可略大，但**不做夸张巨号**（收小以让位数据）。
4. **研究判断 + 双证据区**：研究判断限 1–2 句，紧邻 1–3 条支撑证据与 1–3 条反面证据 / 待验证项；每条都来自结构化内核，优先“指标 + 变化 + 含义”。估值卡为必备区；若卡面空间不足，优先保留研究判断、最高优先级反面证据和来源。
5. **跟踪 / 证伪区**：2–3 项与当前结论直接相关的后续可观测指标，使用等宽小标签 + 正文值；优先填真实重要信息，不用装饰性内容占位。
6. **footer 固定件（锁定）**：墨黑发丝线分隔 → 左 slogan「客观体检 · 不含买卖建议」→ 右 来源简注 / 数据截至（等宽小字）→ 弱化制作来源「由 Rodya-caijing-studio 制作」。完整免责按 `compliance.md`；卡面空间不足时免责随转发文案走，卡上至少保留 slogan + 来源 + 数据截至日 + 制作来源。**不得出现任何个人审核或作者背书。**

## 六、各模块签名图形（定稿规格 2026-07-10）

统一：墨黑网格 + 直角发丝线 + 等宽标签 + tabular 数字 + 「越大越细」；**红 / 绿只标涨跌方向数据，其余一律墨色**；招牌带 + footer 固定件所有模块共用。

1. **基本面 · 护城河五源**：五行清单（无形资产 / 转换成本 / 网络效应 / 成本优势 / 规模效应），每行右侧 ■满 / □空 表 0–2 分 / 源；底部发丝线 + 合计大数（越大越细，如 7 / 10）。主视觉可叠 ROE 头号数（墨色大字）。
2. **财报 · 超预期矩阵**：3 行（营收 / 归母净利 / 扣非）× 列（同比 / 环比 / vs 预期）小表；单元格 ▲＝涨 / 超预期（红）、▼＝跌 / 不及（绿）、＝（墨灰）；beat / miss 文字同色。**全卡唯一常态用红绿处**。
3. **估值 · 高密度分位卡**：先以“历史分位 + 估值状态 + 验证状态”构成结论带，再给满宽墨条；P25 / 中位 / P75 为浅灰刻度，当前位**墨色圆点**（非涨跌，不用红）。分位下紧邻当前估值、同业位置 / 历史中位、隐含预期等 2–3 个关键比较值；随后为 5–8 项指标网格、1–2 句研究判断和“支撑证据 / 反面证据或待验证项”双栏。只能写“估值偏高 / 中性 / 偏低、待验证”等坐标判断，不得接买卖动作。
4. **排雷 · 风险维度**：维度清单（经营 / 财务 / 治理 / 政策 / 流动性）右侧标记 ●重点 / ◐提示 / ○无（**墨色**，不上红绿灯）；底部计数「重点 X · 提示 Y · 无 Z」。总评灯仅专业版，不上客户卡。
5. **产业链 · 利润池图谱**：上中下游三段等宽块，墨色深浅表利润池集中度（越深越集中）+ 各段占比 tabular；底注「墨色越深＝利润池越集中」。
6. **A股打新 · 破发统计**：发行 PE vs 行业 PE 对照（tabular）+ 板块同类组首日破发率大数 + 墨色进度条；纯统计，无申购建议。
7. **港股打新 · 关键因子**：关键因子清单（基石占比 / 孖展超购 / 保荐人近 12 月战绩，中性陈述）+ 同类组首日破发率大数 + 墨条；**对客不含申购建议 / 档位方案**。

签名图形模板落 `assets/`（基本面为参考实现，其余按上表结构套用同一套网格语言）。

## 七、交付与生成

- **先生成 HTML，再按执行端交付。** 生成客户版卡片时，复制 `assets/template-card.html` 里对应模块的 `.card`，把内核数据填进招牌带 / 签名图形 / 关键数字 / 风险 / footer。
- **ChatGPT Codex**：在同一 HTML、同一数据状态下实际捕获 1080×1440 PNG，交付 PNG + HTML + 文案。PNG 是客户直接转发件，HTML 是可复核源文件；不得使用生成式示意图、旧 PNG 或不同版本 HTML 代替实际捕获结果。
- **Claude**：交付 HTML + 文案，并提醒「在浏览器打开后截图即可（3:4 竖屏，手机截屏或 2× 高清均可）」。
- **Codex 例外**：若浏览器或渲染能力被环境阻断，明确报告阻断，降级为 HTML + 文案 + 截图提醒；不得声称已交付 PNG。
- 卡片 HTML 模板：`assets/template-card.html`（v0.5，七模块完整卡片，1080×1440；正文使用固定间距而非均分留白；估值卡含结论带 + 双证据栏；每卡 5–8 项关键数字 + 2–3 项跟踪/证伪指标，配色走锁死 CSS 变量）。
