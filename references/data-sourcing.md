# 取数规范（数据源 + 口径纪律 + 确定性分级）

默认策略：**AI 联网检索/抓取为主，用户上传为补充**（财报截图、终端导出、招股书等）。用户上传的一手材料优先级最高。

硬规则：缺少模块最低一手材料时，按 [data-fallback.md](data-fallback.md) 标注数据档次、结论边界和补充要求。不得把专业版文件降成短评或空框架；但受影响的专业判断、评级、分档或客户版关键卡片必须改为“待核实判断/条件性判断/需补充材料”，不能伪装成强结论。

## 一、数据源优先级

**A股：**

1. 巨潮资讯（cninfo）——定期报告、公告原文（一手）
2. 上交所/深交所官网——公告、监管函
3. 东方财富/同花顺——行情、估值分位、一致预期（二手，需标注）
4. 公司官网投资者关系页

**港股：**

1. 港交所披露易（HKEXnews）——公告、财报、招股书（一手）
2. 公司官网 IR 页
3. 行情源（东财港股/富途公开页等，二手需标注）

**通用：** 行业数据用行业协会/统计局/头部券商公开研报，标明出处。

## 二、口径纪律（七条）

1. **关键数字必标来源**：文内脚注或括注，格式 `（来源：2025年报，巨潮）`。
2. **区分"已核实"与"估算"**：原文核实过的标"已核实"；推算/二手引述标"估算"并写明假设（如 WACC 粗估）。
3. **标时效**：每份输出标"数据截至 YYYY-MM-DD"；行情类数据精确到日。
4. **会计准则差异**：A股 CAS vs 港股 HKFRS/IFRS，涉及跨市场对比时提示口径差异（如金融资产分类、投资物业计量）。
5. **币种**：RMB / HKD 全文统一标注，跨币种对比给出汇率与取值日。
6. **财报频率**：A股季报、港股以半年报为主——同比环比的可比性受此约束，横向对比时说明。
7. **冲突数据并列**：不同来源数字打架时，并列呈现 + 标注各自来源，不臆断取舍。

## 三、确定性分级（每份输出整体标注一档，关键数字逐条标注）

| 级别 | 定义 | 例 |
|---|---|---|
| A | 一手原文核实（定期报告、公告、招股书） | 年报里的营收、审计意见 |
| B | 权威二手（交易所数据、主流行情终端、公司 IR 确认） | 估值分位、一致预期 |
| C | 估算/推断（有明确假设的计算，或无法溯源的引述） | 粗估 WACC、媒体转述的订单额 |

C 级数据不得作为结论的唯一支撑；若结论主要靠 C 级数据，结论本身标为"待核实线索"或"条件性判断"。

## 四、数据缺口声明

没拿到的关键数据（如未披露分部毛利），在输出末尾如实列出，写进内核 `数据缺口` 字段。宁可留白，不可编数。

## 五、数据缺口执行

出现以下任一情形，立即读取 [data-fallback.md](data-fallback.md)：

- 无法取得年报、财报、招股书、发行公告等核心一手材料。
- 行情、估值、破发率、孖展数据不能确认截至日。
- 不同来源关键数字冲突，且无法统一口径。
- 用户只提供截图或转述，无法追溯原文。
