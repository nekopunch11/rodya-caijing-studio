# 财经·排雷（caijing-risk）

**只咬一个问题：有没有雷。**只找雷、不评质地——质地好坏是 `caijing-fundamental` 的事。

**边界（接力关系）**：基本面全面体检 → `caijing-fundamental`；估值风险（贵）不算雷 → `caijing-valuation`。

执行前先读共享层：`../references/compliance.md`、`../references/output-spec.md`、`../references/data-sourcing.md`。

## 一、四类雷分类（内核的组织骨架）

| 类别 | 典型形态 |
|---|---|
| 财务雷 | 利润操纵、收入造假、资产虚增 |
| 治理雷 | 大股东资金占用、违规担保、关联输送 |
| 经营雷 | 主业恶化、大客户依赖断裂、政策冲击 |
| 交易结构雷 | 高比例质押、集中解禁、持续减持 |

## 二、分析框架（五个分析块）

### 1. 学术模型体检（结果标"估算"级，给评分一个可引用出处）

- **Beneish M-Score**（盈余操纵概率，8 因子）：> -1.78 进入预警区。注明对中国市场的适配局限（样本为美股，作预警参考非定罪）。
- **Altman Z-Score**（财务困境）：制造业用原版，非制造业用修正版；报告分区间（安全/灰色/困境）。

### 2. A股本土红旗库（逐项扫描，命中必须给公告/报表原文证据）

| 红旗 | 检查口径 |
|---|---|
| 大存大贷 | 货币资金与有息负债同时高企（康美/康得新式） |
| 非标审计意见 | 保留/无法表示/否定意见，及带强调事项段 |
| 其他应收款异常 | 占总资产比例畸高或激增（资金占用嫌疑） |
| 在建工程久不转固 | 长期挂账不转固定资产（利润调节嫌疑） |
| 频繁更换审计师/财务总监 | 近 3 年更换记录 |
| 大股东高比例质押 | 控股股东质押率、平仓压力 |
| 违规担保与重大诉讼 | 公告与年报附注 |
| 商誉/净资产比 | 减值悬顶 |
| 存货应收异动 | 增速远超收入且无行业解释 |
| 减持与解禁时间表 | 未来 12 个月解禁量、近期重要股东减持 |
| 监管记录 | 问询函、警示函、立案调查 |

### 3. 港股专属红旗库

老千股行为史（频繁合股、大比例供股/配售摊薄）；CCASS 货源集中度（高度集中易操纵）；核数师规模与更换记录；长期停牌史。

### 4. 四类雷汇总

将命中红旗归入四类雷框架，逐条给证据与严重度。

### 5. 评分与总评灯

- **一票否决项**（命中任一直接红灯，不进加权）：非标审计意见（保留及以上）、被立案调查、违规担保未解除、老千股行为史。
- 其余按严重度加权 → **红/黄/绿**三档总评。
- 每面旗必须有证据，无证据不挂旗；证据不足写进"待核实线索"。

## 三、合规特别注意（本模块从严，有法律风险）

只陈述公告与报表**事实**（"2024 年报被出具保留意见（来源：年报，巨潮）"），**禁用**"造假、骗子、必爆、庞氏"等定性/煽动词。客户版更收敛：只列红旗事实清单 + "需关注"，不做爆雷预言。M/Z 模型结果表述为"进入统计预警区间"，不是"该公司在操纵利润"。

## 四、取数清单

年报全文尤其附注（审计意见、其他应收款、担保诉讼、关联交易）；质押数据（中登/交易所）；解禁减持时间表；监管函件记录；港股加 CCASS 持仓、公司行动历史（合股/供股）。

## 五、内核字段（继承通用 schema）

`关键数字` 固定 7 条：M-Score、Z-Score（含区间）、红旗命中数（红/黄分列）、控股股东质押率、商誉/净资产、其他应收款占比、总评灯色。

`结论.客观概括` 句式："{公司}红旗扫描命中 X 项（红 N/黄 M），一票否决项{无/有：…}，总评{绿/黄/红}。"

## 六、输出

### 专业版 docx（2000–3000 字）

1. 一页概要（总评灯 + 一票否决核查结果 + 关键数字表）
2. 学术模型体检
3. 红旗逐项扫描（A股库或港股库）
4. 四类雷汇总
5. 数据附录（证据清单）+ 免责声明

### 客户版（卡片 + 文案）

- **红黄绿评分卡**：总评灯 + 命中红旗清单为主视觉，布局见 card-components.md。
- **文案模板**（150–300 字）：

> 【{公司} 风险体检】基于公开披露信息：{事实 1（来源）}；{事实 2（来源）}。风险扫描共提示 N 项需关注事项，整体评级{绿/黄/红}。以上为客观数据整理与风险揭示，供参考。
> （固定免责声明）

## 七、工作流

1. 一票否决项先查（快速止损）→ 全量取数。
2. 模型 + 双库扫描，产出结构化内核（每旗带证据）。
3. 渲染专业版 docx + 客户版卡片文案。
4. 合规检查（措辞从严，逐句过）。
5. 交付，标数据截至日 + 确定性分级 + 待核实线索。
