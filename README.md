# 📊 财经内容台 · rodya-caijing-studio

> 给个人投资者、券商投顾 / 客户经理使用的 A股 + 港股财经分析与内容生产工具包。
> 个人投资者、研究人员和券商投顾 / 客户经理都获得同一套专业研究内核；需要转客户时，再从同一内核生成保留研究判断、但不含交易动作的客户合规版。
> 它不是荐股器，也不是情绪化喊单模板，而是一套把“研究判断、数据来源、风险提示、合规表达”放在一起的财经工作台。

## 🧭 为什么做这个

财经分析最难的地方，往往不是“写几句观点”，而是把一家公司、一份财报、一个题材或一只新股，拆成可检查、可复用、可交付的分析流程。

个人投资者需要的是：别被标题党、目标价和市场情绪牵着走。看一家公司，先知道它的钱从哪来、财报质量怎么样、估值在什么位置、有没有明显风险、后面哪些指标会证伪原来的判断。

券商投顾 / 客户经理需要的是：每天都要跟客户沟通，但不能靠“闭眼买”“上车”“目标价”这种话术。真正能长期使用的内容，必须有数据、有来源、有风险提示、有合规边界，还要能快速做成客户看得懂的卡片和文案。

所以这套工具做两件事：

- **自用分析**：把基本面、财报、估值、排雷、产业链、A股打新、港股打新拆成七个标准模块，帮你形成一套稳定的研究检查表。
- **客户合规交付**：同一份结构化分析内核可以生成客户版 docx、卡片、文案或 PPT；保留证据化研究判断、来源、边界和风险，去掉买卖/申购动作、目标价、收益承诺和个性化资金建议。

它解决的不是“替你拍脑袋做决策”，而是让你每次分析都有框架、每次输出都有依据、每次对外表达都有边界。

## 🧩 七个模块，各咬一个问题

### 🩺 财经·基本面 `caijing-fundamental` —— 这公司好不好

三问主线：钱从哪来、是不是真钱、还能赚多久。先定义行业真正跑的关键指标，再用杜邦三因子、ROIC–WACC、现金流含金量、Morningstar 五源护城河、可持续增长率和阶段坐标做全面体检；进一步加入管理层目标兑现、资本配置记分卡、单位经济模型与正常化盈利，并沉淀 3–5 条带证伪条件的投资逻辑支柱。

### 📈 财经·财报 `caijing-earnings` —— 这季财报行不行

超预期判定采用三层基准体系：分析师一致预期 → 公司业绩指引 → 历史季节性推算。除点估计外，还看覆盖人数、预期分布与 30/60/90 天预测修正；用 reported→adjusted 桥拆股权激励、减值、公允价值等调整项，并回看近 4–8 期指引兑现率。合同负债、订单和资本开支构成前瞻指标，本期结果继续映射投资逻辑支柱。

### ⚖️ 财经·估值 `caijing-valuation` —— 现在贵不贵

行业自适应估值锚：周期股规避低 PE 陷阱，金融股 PB-ROE 联动，未盈利成长股使用 PS/EV-Sales。先构建正常化盈利与完全摊薄股本，再做历史分位、同业四分位和反向 DCF；DCF 强制披露终值占比，多元公司执行 SOTP，最后用至少两类方法交叉验证。专业版做三情景敏感性，但不输出目标价。

### 🚨 财经·排雷 `caijing-risk` —— 有没有雷

M-Score、Z-Score 与 A/港股红旗库共同筛查，但模型只作线索排序：每个信号都要说明原始适用范围、本土局限、可能误报并回到附注/审计/监管证据验证。审计事项拆成意见类型、涉及金额、持续期间、整改状态与实际影响；客户合规版保留风险方向判断，同时呈现证据、边界和缓释因素。

### 🔗 财经·产业链 `caijing-industry` —— 题材链条怎么分、代表公司有哪些

先定义题材真正跑的行业指标，再用渗透率 S 曲线标定生命周期。利润池用价值量、单位利润、集中度、议价权、供给约束五维证据评分，不做伪精确乘积；代表公司再拆业务纯度、收入弹性、利润弹性、竞争优势与估值已反映程度。适用题材可启用瓶颈点方法，客户合规版保留利润沉淀与供给约束判断，但不做推荐排序。

### 🆕 财经·A股打新 `caijing-ipo-a` —— 新股破发统计风险多大

注册制新股客观体检：发行 PE 行业偏离与可比公司四分位、超募、流通盘、板块分组破发率、网下报价和弃购率。破发率同时给样本数、基准率与 Wilson 置信区间，并滚动复盘档位相对板块基础破发率是否有增量价值——仍是历史样本参考，不是申购建议。

### 🎯 财经·港股打新 `caijing-ipo-hk` —— 怎么打、打多少 + 破发统计风险

两层响应。默认研究层覆盖保荐人、基石、旧股、绿鞋、同业估值、绝对孖展金额、18A/18C 和同类组统计；明确问打多少时才进入行动顾问层，按实际/可比申请档位、风险预算和用户资金参数给条件性方案。客户合规版保留发行质量与统计风险判断，不含申购、融资或处置动作。

> 模块之间有接力关系：基本面里估值和风险只做简版，想深挖时估值、排雷模块接着上，不重复劳动。

## 📦 每个模块出什么

| 模块 | 专业研究版（默认） | 客户合规版（按需） |
|---|---|---|
| 🩺 基本面 | 研报级体检报告 docx | 体检雷达卡 + 文案 |
| 📈 财报 | 财报点评 docx | 财报速览卡 + 文案 |
| ⚖️ 估值 | 估值分析 docx | 分位 band 卡 + 文案 |
| 🚨 排雷 | 排雷报告 docx | 风险提示卡 + 文案 |
| 🔗 产业链 | 产业链梳理 docx | 图谱卡 + 文案 |
| 🆕 A股打新 | 新股体检 docx | 打新速览卡 + 文案 |
| 🎯 港股打新 | 新股体检 docx + 自用申购方案（三档+推荐） | 打新速览卡 + 文案 |

> 单模块和多模块默认均不自动生成 PPT；只有用户明确点名“汇报/PPT/deck/幻灯”时才生成。单模块默认只出专业研究 docx；用户说“转客户/客户版”但未指定形态时默认生成卡片+文案，也可点名客户版 docx 或 PPT。所有形态同源一份结构化分析内核。

## 🚀 安装

本包同时兼容 Claude 和 Codex——两家用的是同一套 [agent skills 开放标准](https://agentskills.io)，一份文件两边通用。

### 方式一：让 AI 自己装（最省心）

不想碰命令行？把仓库地址直接发给你的 Claude 或 Codex，加一句话：

```
帮我把这个 skill 装到你的 skills 目录：
https://github.com/nekopunch11/rodya-caijing-studio
```

它会自己 clone 到正确位置。装完可以让它报一遍七个模块的名字，对得上就是装好了。

### 方式二：npx 一行命令（全平台通用）

```bash
npx skills add nekopunch11/rodya-caijing-studio
```

[skills CLI](https://github.com/vercel-labs/skills) 是 agent skills 生态的通用安装器，自动检测机器上的 Claude Code / Codex / Cursor 等 agent 并装进对应目录。加 `-g` 装全局；日后 `npx skills update` 一键更新。

### 方式一：让 AI 自己更新

把这句话发给 Claude 或 Codex：

```
帮我把已安装的 rodya-caijing-studio skill 更新到最新版：
```

让它检查本地 skills 目录里的 `rodya-caijing-studio`，拉取 GitHub 最新版本并覆盖旧文件。更新后可以让它报一遍 README 里的安装与更新小节，确认已经是新版。

如果需要备份，放到 `~/.codex/skill-backups/`，或把备份里的 `SKILL.md` 改名为 `SKILL.md.disabled`。更新完成后检查 `~/.codex/skills/` 下是否只剩一个 `rodya-caijing-studio`。

## 💬 用法

装好后直接说人话，父 skill 自动路由到对应模块：

- 🩺 **基本面**："全面分析一下宁德时代"——默认拿到完整专业研报；需要转客户时再生成保留研究判断的客户合规版。
- 📈 **财报**："茅台出年报了，点评一下"——财报季批量快评，每家一张速览卡，客户群里一天可以专业地出现很多次。
- ⚖️ **估值**："招商银行现在贵不贵"——"能不能买"这种问题也可以丢给它，它只答贵不贵，判断权留给客户。
- 🚨 **排雷**："帮我排查下这家公司有没有雷"——客户拿着小道消息来问时，用红旗清单和公告证据代替拍胸脯。
- 🔗 **产业链**："梳理一下固态电池产业链"——题材热起来时给客户一张图谱，哪些环节和公司真实相关，一目了然。
- 🆕 **A股打新**："XX 明天科创板申购，分析下"——申购日前出体检，给历史样本下的统计风险提示档位。
- 🎯 **港股打新**："分析这只港股新股"默认先做统一专业体检；明确问“打多少”时，再按档位配售证据、风险预算和个人资金参数进入行动顾问模式。
- 📽️ **组合出 PPT**："基本面 + 估值 + 排雷，出个汇报 PPT"——晨会和客户沙龙的主场。
