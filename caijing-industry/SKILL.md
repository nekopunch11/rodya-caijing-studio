# 财经·产业链（caijing-industry）

**只咬一个问题：题材链条怎么分、代表公司有哪些。**这是题材层模块——梳理的是链条和格局；链上单个标的的质地/估值/风险，接力给个股模块。

**边界（接力关系）**：单个标的深挖 → `caijing-fundamental` / `caijing-valuation` / `caijing-risk`。

执行前先读共享层：`../references/compliance.md`、`../references/compliance-rendering.md`、`../references/output-spec.md`、`../references/data-sourcing.md`、`../references/data-fallback.md`。

## 一、主线：从"沾边"到"受益"的三次过滤

1. **题材是真是虚**——驱动与生命周期定位
2. **钱沉淀在哪个环节**——利润池分析
3. **谁是真受益**——纯度指标过滤

## 二、分析框架（五个分析块）

### 1. 题材界定与驱动

题材边界一句话；驱动类型（政策 / 技术 / 需求，可多选）；催化时间表（已落地 vs 预期中，分开列）。

### 2. 生命周期定位（渗透率 S 曲线）

当前渗透率数据（标来源与口径）→ 定位阶段：**0→1**（主题期：叙事驱动、业绩未兑现）或 **1→N**（兑现期：看订单和收入）。这是对"题材温度"的客观标定，不是买卖时钟。

### 3. 上中下游图谱

环节拆解（上游材料/设备 → 中游制造/集成 → 下游应用/渠道）；各环节价值量占比（单位产品成本/价值拆分，标注来源级别）。

### 4. 利润池分析（钱最终沉淀在哪）

每环节 = 价值量占比 × 竞争格局（CR3/CR5、玩家数量、国产化率）→ 高价值量 + 高集中度的环节才是利润沉淀区；高价值量 + 完全竞争 = 增收不增利。**"受益"≠"沾边"**，这是本模块的核心判别。

### 5. 代表标的清单（纯度过滤）

每环节代表公司表，字段：公司/代码、环节地位（龙头/主要玩家/新进入者）、**纯度**（相关业务收入占比——年报分部数据为准 A 级，无披露则估算标 C 级）、按纯度降序排列。纯度 <10% 的单独归入"低相关（题材关联度弱）"，明确标示。

## 三、合规特别注意

标的清单是**客观图谱**，不是股票池：按业务相关度客观列示，不做推荐排序、不加"首选/受益最大"类字眼（专业版可说"利润池主要沉淀在 X 环节"，客户版只呈现数据）。清单后固定加一句："以上标的按业务相关度客观列示，不构成推荐。"

## 四、取数清单

行业协会/统计局/工信部等官方口径（渗透率、市场空间）；头部券商公开研报（图谱与价值量拆分，标 B/C 级）；各标的年报分部收入（纯度用，一手 A 级）；产业政策原文（驱动核实）。

## 五、内核字段（继承通用 schema）

`关键数字` 固定 6 条：当前渗透率及阶段、市场空间（如有权威口径，注明出处）、核心环节 CR3/CR5、利润沉淀环节、纯度 Top3 标的及占比、催化时间表条数（已落地/预期中）。

`结论.客观概括` 句式："{题材}当前渗透率约 X%（{0→1/1→N} 阶段），链条 N 个环节中利润主要沉淀在{环节}，纯度最高的代表标的为{…}（相关收入占比 Y%）。"

## 六、输出

### 专业版 docx（2500–4000 字，题材层信息量大取上限）

1. 一页概要（客观概括 + 专业判断 + 关键数字表）
2. 题材驱动与生命周期
3. 上中下游图谱（配图谱表）
4. 利润池分析
5. 代表标的清单（纯度表）
6. 数据附录 + 免责声明

### 客户版（卡片 + 文案）

- **图谱卡**：上中下游环节 + 利润池深浅标记 + 各环节代表公司示例（带纯度或“未披露，待核实”），布局见 card-components.md。
- **文案模板**（150–300 字）：

> 【{题材} 产业链速览】{题材}当前渗透率约 X%，处于{阶段描述}。产业链分{上/中/下游简述}，其中{环节}环节集中度较高（CR3 约 Y%）。代表公司示例包括：{公司 A}（相关收入占比 a%）、{公司 B}（b%）。以上公司按业务相关度客观列示，不构成推荐。需关注：{风险 1}。
> （固定免责声明）

## 七、工作流

1. 定题材边界 → 取数（官方口径优先），逐条标来源与确定性；缺少渗透率、产业口径或分部收入时按 data-fallback.md 降级。
2. 依五个分析块产出结构化内核（纯度证据链完整）。
3. 渲染专业版 docx + 客户版卡片文案。
4. 合规检查（清单表述从严，客户版按 compliance-rendering.md 降级）。
5. 交付，标数据截至日 + 确定性分级 + 数据缺口。
